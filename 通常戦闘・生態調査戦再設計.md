# 『封域異獣録』通常戦闘・生態調査戦 再設計

版：0.1  
更新日：2026年7月16日  
状態：実装前設計

## 1. 再設計の目的

『封域異獣録』の基本体験を、次の一文で説明できる状態へ戻す。

> 異獣を仲間にし、それぞれの特技を組み合わせて戦い、観察によって特別な協力方法を見つけるモンスター育成RPG。

現状は、最初の接触から観察・防御・浄化・鎮静・協力要請という固有手順を扱うため、次の問題がある。

- 異獣自身の特技より、画面に示された攻略手順が主役になっている。
- 場面ごとに新しい操作語が増え、説明を追加するほど画面が混雑する。
- 通常戦闘と特殊な生態調査の違いを体験する前に、特殊ルールから学ばせている。
- プレイヤーが「自分の仲間は何ができるのか」を記憶しにくい。
- おすすめ作戦だけを押すと進められる一方、自分で戦術を選んだ実感が弱い。

この問題は文言や配置だけでは解決しない。戦闘の基本動詞と、最初に教える順序を変更する。

## 2. 採用する二層構造

戦闘を「通常戦闘」と「生態調査戦」に分ける。ただし別々のゲームにはせず、同じ戦闘画面と同じ異獣コマンドを使用する。

| 項目 | 通常戦闘 | 生態調査戦 |
|---|---|---|
| 主な対象 | 一般的な野生異獣、汚染個体、遺跡防衛個体 | 初遭遇の重要個体、救助対象、地域ボス |
| 主目的 | 敵対個体を退かせる | 個別の生態目的を達成する |
| 基本操作 | たたかう・特技・防御・交代 | 通常操作＋主人公の調査行動 |
| HPが0 | 戦闘不能になり、戦闘から離脱 | 対象が逃走または救助失敗になる場合がある |
| 観察 | 任意。有利情報や協力条件を得る | 成功経路の発見に重要 |
| 協力要請 | 種ごとの簡潔な条件達成後 | 物語・生態条件の達成後 |
| 説明量 | 最小限 | 目的と未達成条件だけ追加 |

### 2.1 通常戦闘の役割

通常戦闘は、異獣の技を使う楽しさ、役割分担、成長報酬を担当する。

- 攻撃しても調査思想に反しない。HPは戦闘継続力であり、0は死亡ではなく戦闘不能・撤退を表す。
- 観察を使わなくても勝利できる。
- 勝利後に経験値、素材、異獣録の基礎情報を得る。
- 既知の協力条件を満たした場合は、倒し切る前に協力要請できる。

### 2.2 生態調査戦の役割

生態調査戦は、本作固有の研究・救助・理解の楽しさを担当する。

- 画面上部に「救助調査」「重要個体」「地域中枢」などの種別を明示する。
- 目的は一文で表示する。例：「濾過膜の汚染を除き、スミワタリを鎮める」
- 通常の異獣コマンドを取り上げない。
- 主人公の観察によって、未知の条件を一つずつ明らかにする。
- 正解手順そのものは常時表示せず、判明済み条件と直前の結果だけ表示する。

## 3. プレイヤーが覚える基本動詞

異獣の行動は、全戦闘で次の4つに固定する。

1. **たたかう**：その種の基礎攻撃を使い、活性を回復する。
2. **特技**：装備した最大4個から1つを選び、活性を消費する。
3. **防御**：受ける被害を減らし、活性を多く回復する。
4. **交代**：前衛と控えを入れ替える。

主人公は異獣とは別に、1ラウンド1回まで次の「調査行動」を行える。

- 観察
- 道具
- 協力要請
- 何もしない

観察は異獣の特技と混ぜない。プレイヤーは「仲間は戦う、主人公は調べて支援する」と役割を分けて理解できる。

画面上では主人公行動を一つの「調査」ボタンにまとめ、タップ後に観察・道具・協力要請を開く。異獣の4コマンドと同時に7〜8個のボタンを並べない。

## 4. 特技を理解しやすくする規則

### 4.1 一個体の初期情報量

加入直後の異獣は、基礎攻撃と2個の特技から始める。最大4枠は成長後に使う。

| 異獣 | 役割ラベル | たたかう | 初期特技1 | 初期特技2 |
|---|---|---|---|---|
| トモシゴケ | 回復・鎮静 | 灯角突き | 灯苔の雫 | 静かな明滅 |
| ヌマクグリ | 防御・かばう | 大尾打ち | 身代わり潜行 | 泥幕 |
| スミワタリ | 浄化・回復 | 水圧弾 | 澄み流し | 2個目は加入後の成長で解放 |

開始時点で3体×4技能を暗記させない。最初の2体は各2特技、加入したスミワタリは1特技から教える。

### 4.2 表示形式

特技は、名称だけで意味を推測させず、常に次の順で表示する。

```text
灯苔の雫
味方1体を回復　活性25
```

必要な情報は次の4点に限定する。

- 名前
- 対象
- 即時効果
- 活性消費

計算式、世界設定、詳細な状態相互作用はタップ後の詳細または異獣録へ分離する。

### 4.3 役割を動作で覚えさせる

説明文ではなく、最初の戦闘で役割が必要になる状況を作る。

- トモシゴケ：傷ついた味方へ「灯苔の雫」を使う。
- ヌマクグリ：予告攻撃に「防御」または「身代わり潜行」を使う。
- スミワタリ：加入直後の汚染戦で「澄み流し」を使う。

一度使った特技には短い結果を表示する。

```text
灯苔の雫 → トモシゴケのHPが24回復
```

## 5. 通常戦闘の1ラウンド

1. 敵の構えや予告行動を表示する。
2. 主人公の調査行動を任意で選ぶ。
3. 前衛を一体ずつ選び、4コマンドから行動を決める。
4. 全員の予定を短い一覧で確認する。
5. 行動を演出付きで解決する。
6. HP・活性・状態変化を数値と短文で表示する。
7. 次ラウンドへ進む。

### 5.1 入力短縮

- 一度選んだ行動は、次の異獣へ自動でフォーカスを移す。
- 対象が一つしかない場合は対象選択を省略する。
- 前ラウンドと同じ指示を使う「くり返す」を用意する。
- チュートリアル後は「おまかせ」を任意で使える。
- おすすめ作戦は常設の主ボタンにせず、小さな「ヒント」または初回限定の強調にする。

プレイヤーが実際の「たたかう」「特技」「防御」を押すことで、操作と結果を結び付ける。

## 6. 通常の仲間化

通常個体の仲間化は、複雑な複数手順にしない。種ごとに協力条件を一つだけ持つ。

例：

- HPを半分以下にする。
- 鎮静状態にする。
- 特定属性の攻撃を使わず2ラウンド経過する。
- 汚染を解除する。

初遭遇では、主人公の「観察」で条件を公開する。調査済みの種は戦闘開始時から表示する。条件達成後の協力要請は従来どおり確定成功とし、確率抽選にはしない。

画面では条件を一行だけ表示する。

```text
協力条件　鎮静状態にする　未達成
```

条件を満たすと、主人公コマンドの「協力要請」が光る。条件一覧や5段階の手順は、通常戦闘では表示しない。

## 7. 最初の10分の再構成

### 7.1 出発前ブリーフィング

「同行する異獣を確認する」という必須操作は削除する。編成を選べず、新情報も確定しないため、ゲーム上の判断になっていない。

ブリーフィングは次の一画面にする。

- 任務：灰苔湿原の異常と先遣隊記録を調べる。
- 同行：トモシゴケ「回復役」、ヌマクグリ「防御役」。
- 主ボタン：「灰苔湿原へ出発する」
- 任意リンク：「特技を見る」

「特技を見る」を開いた場合だけ、各個体の初期2特技を表示する。閲覧しなくても出発できる。

### 7.2 初回通常戦闘

スミワタリ救助の前に、若いキリハネとの短い通常戦闘を1回置く。既存のキリハネ用素材と技能の一部を使い、チュートリアル用にHPと行動を単純化する。新しい異獣種と専用素材は増やさない。

教える順序：

1. トモシゴケの「たたかう」で、基礎攻撃と活性回復を教える。
2. ヌマクグリの「防御」で、被害軽減と活性回復を教える。
3. キリハネは第1ラウンドを必ず耐え、トモシゴケへ軽い攻撃を行う。第2ラウンドに「灯苔の雫」を提示する。
4. 敵を戦闘不能にして勝利する。
5. 経験値と素材を得る。

初回は各コマンドを短く強調するが、正解作戦を一括実行しない。プレイヤー自身が実際のコマンドを押す。

### 7.3 スミワタリ救助調査

通常戦闘を一度経験した後、スミワタリを「救助調査」として提示する。

画面上部：

```text
救助調査
目的：濾過膜の汚染を除き、スミワタリを鎮める
```

戦闘下部の異獣コマンドは通常戦闘と同じにする。追加するのは主人公の調査行動と、判明済み条件だけにする。

想定手順：

1. 主人公が観察し、「汚染除去」と「鎮静」が必要だと判明する。
2. ヌマクグリが防御または身代わりで処置時間を作る。
3. 主人公が汚染除去具を使う。
4. トモシゴケが既に覚えた「静かな明滅」を使う。
5. 主人公が協力要請する。

表示するのは現在の目的、判明した条件、直前の結果だけとする。5手すべてを最初から手順一覧にしない。

### 7.4 加入直後

スミワタリ加入時は「澄み流し」を一つだけ大きく見せる。

```text
新しい仲間　スミワタリ
役割：浄化
特技：澄み流し　味方の汚染を解除して少し回復
```

次の通常戦闘に汚染攻撃を持つ敵を置き、すぐに澄み流しを使わせる。加入演出と実用体験を連続させる。

## 8. 戦闘画面の情報構成

390px幅では、通常時に見せる情報を次の順へ限定する。

1. 敵の姿、名前、HP、次の行動予告
2. 選択中の味方の名前、HP、活性、状態
3. たたかう・特技・防御・交代
4. 主人公の調査行動
5. 他の前衛を示す小型タブ

常時表示しない情報：

- 長い目的説明
- 全行動ログ
- ダメージ計算式
- 全協力条件
- おすすめ作戦の詳細一覧
- 世界設定の説明

生態調査戦だけ、敵表示の直下へ一行の目的と条件チップを追加する。

### 8.1 意味のある操作だけを必須にする

- 閲覧・確認だけのボタンで進行を止めない。
- 必須操作にするのは、戦闘行動、編成変更、選択、資源使用など結果が変わるものだけとする。
- 説明を読む操作は任意にし、必要な知識は直後の行動結果からも学べるようにする。
- チュートリアルで強調するのは、以後も繰り返し使う共通コマンドだけとする。

## 9. 演出で説明を減らす

ゲーム感は文章ではなく、入力に対する反応で作る。

- 攻撃：対象が揺れる、HPゲージが減る、短い打撃音。
- 回復：緑青の光、回復値が浮く、HPゲージが戻る。
- 防御：盾・殻のエフェクト、被害値の横へ「軽減」。
- 鎮静：敵の動きと彩度が落ち、警戒アイコンが縮む。
- 浄化：黒い汚れが剥がれ、状態アイコンが消える。
- 協力成立：戦闘曲を止め、相手が味方側へ移動する。

一つの行動結果を「動き・数値・6〜20文字の短文」の三つで伝える。長文説明は使わない。

## 10. データ設計方針

現在の地点別専用パネルを一度に置き換えず、共通戦闘へ段階的に寄せる。

```ts
type EncounterKind = 'normal' | 'ecological' | 'boss'

type EncounterDefinition = {
  id: string
  kind: EncounterKind
  objective: string
  victoryRule: string
  recruitmentRule?: RecruitmentRule
  tutorial?: TutorialDefinition
}

type BattleSkillDefinition = {
  id: string
  name: string
  category: 'attack' | 'heal' | 'guard' | 'cleanse' | 'calm' | 'field'
  target: string
  vitalityCost: number
  shortEffect: string
}
```

共通化するもの：

- 味方のHP・活性・状態
- たたかう・特技・防御・交代
- 行動予定とラウンド解決
- 短い結果表示
- 主人公の調査行動

遭遇ごとに定義するもの：

- 勝利・成功条件
- 協力条件
- 敵の行動パターン
- 固有の環境状態
- チュートリアルの強調箇所

## 11. 既存仕様から維持するもの

- 前衛3体＋控え3体
- 個体別の活性値
- 最大4技能の装備枠
- 主人公の観察・道具・協力要請
- 条件達成後の協力要請は確定成功
- 状態付与は条件を満たせば原則成功
- 非消失型の共鳴継承
- 生態図鑑風2Dイラスト

変更するもの：

- 最初に生態調査戦を教える順序
- 「基礎行動」というUI名称を「たたかう」へ変更
- 観察を通常戦闘の必須手順から任意の主人公行動へ変更
- おすすめ作戦を主操作から補助的なヒントへ変更
- 通常戦闘の協力条件を一種一条件へ簡略化

## 12. 実装順

### 第1段階：基本動詞を見せる

1. 出発前の必須確認を削除する。
2. ブリーフィングへ役割ラベルと任意の特技表示を追加する。
3. 共通の「たたかう・特技・防御・交代」UIを作る。
4. トモシゴケとヌマクグリの初期特技を表示できるようにする。

### 第2段階：通常戦闘を先に体験させる

1. 小型の通常敵と短い初回戦闘を追加する。
2. 攻撃、防御、回復を実際のコマンドで教える。
3. 勝利、経験値、素材の小さな報酬画面を追加する。

### 第3段階：スミワタリ救助を共通UIへ移す

1. 通常戦闘UIへ救助目的と調査条件を追加する。
2. 観察・道具・協力要請を主人公行動として接続する。
3. 既存の5ボタン式チュートリアルを廃止する。
4. 加入後すぐに澄み流しを使う通常戦闘を置く。

### 第4段階：既存地点を移行する

1. 観測櫓跡を、通常戦闘＋任意のキリハネ協力へ変更する。
2. 沈み水路を、汚染状態を学ぶ通常戦闘へ変更する。
3. 浄化施設中枢を、生態調査戦またはボス戦として共通化する。

## 13. プレイテストの合格条件

説明文を読ませた量ではなく、プレイヤーの理解と選択で判定する。

- 出発前に意味のない確認操作がない。
- 初回通常戦闘を、一画面一文以内の案内で完了できる。
- 初回戦闘後に「トモシゴケは回復」「ヌマクグリは防御」と答えられる。
- 通常戦闘は観察を使わなくても勝てる。
- 特技選択時に、名前・対象・効果・消費がスクロールなしで見える。
- スミワタリ戦が通常戦闘とは異なる「救助調査」だと開始5秒以内に分かる。
- スミワタリ加入後に「澄み流しが何をする技か」を答えられる。
- 主要操作中、画面に同時表示する主ボタンは4個以内とする。
- おすすめ作戦を使わず、自分で選んだ操作として戦闘を完了できる。

## 14. 今回は変更しないもの

- タイルマップ自由移動への変更
- 共鳴継承の詳細実装
- 育成数値とレベル曲線の全面調整
- すべての異獣への新規技能追加
- 高度な自動戦闘AI
- 本格的な効果音・アニメーション制作

まず、通常戦闘で異獣の役割と特技を楽しめるかを検証する。その結果を確認してから、探索形式や演出素材の規模を判断する。

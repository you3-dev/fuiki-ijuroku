# 『封域異獣録』観測櫓跡・汎用戦闘実装記録

更新日：2026年7月13日

対象：`app/`

判定：前衛3体＋敵1体の汎用戦闘縦切りと、観測櫓跡の地点完結まで成立

## 1. 実装したプレイ範囲

```text
観測櫓跡へ到着
→ キリハネの霧中戦
→ 鳴き声を観察
→ 鎮静音具と静かな明滅で応答
→ 協力要請
→ キリハネを控えへ登録
→ 先遣隊第1記録を回収
→ 反射板を合わせて上流弁復旧
→ 研究所へ帰還
```

帰還時点で、先遣隊記録は `1/3`、水路弁復旧は `1/2`、異獣録は `4/6`になる。

## 2. 汎用戦闘モデル

今回の戦闘モデルは次の情報をセーブへ含める。

- ラウンド番号と保存可能な乱数状態
- 前衛3体と敵1体の現在HP、活性値、防御状態
- 主人公の調査支援
- 各前衛の予定行動、技能、対象
- 霧中の残りラウンド、警戒度、観察・応答・鎮静条件
- 戦闘ログと終了理由

前衛の選択肢は、基礎行動、防御、種を代表する技能1個に限定した。

| 個体 | 基礎行動 | 技能 |
|---|---|---|
| トモシゴケ | 灯角突き | 静かな明滅 |
| ヌマクグリ | 大尾打ち | 身代わり潜行 |
| スミワタリ | 水圧弾 | 澄み流し |

基礎行動は活性値 `+20`、防御は `+25`。技能はマスターデータの消費値を使う。単一敵への基礎行動は対象を自動選択し、身代わり潜行と澄み流しは味方対象を選択する。

行動順は優先度、機敏、個体IDの順で決める。今回の優先度は、身代わり潜行、防御、静かな明滅、回復、敵行動、基礎行動の順である。同じセーブ状態と予定行動からは同じ被害とログを再現する。

## 3. 協力条件

推奨手順は3ラウンドである。

1. 主人公が霧中の鳴き声を観察し、前衛3体は全員防御する。
2. 主人公が鎮静音具で周期を返し、トモシゴケが静かな明滅を使う。必要に応じてヌマクグリが身代わり、スミワタリが回復する。
3. `鳴き声観察＋周期応答＋鎮静＋警戒度20以下`で協力要請する。

条件達成後の協力要請は抽選なしで成功し、残りの予定行動と敵行動を実行せず終了する。

## 4. 確定保存と復帰

ラウンド処理を次の二段階へ分けた。

1. `commitRound` で主人公支援、前衛3体の予定、対象、乱数状態を確定保存する。
2. 保存成功後に `resolveRound` で行動を一度だけ解決する。

アプリ終了が二段階の間に起きた場合、`phase: committed` の状態から同じラウンドを再開できる。解決済みの状態に `resolveRound` を再実行しても変化しない。

敵HPを0にした場合と前衛全滅時は地域報酬を付与せず、遭遇直前から再試行する。通常の低損失チェックポイントとして実装した。

## 5. 加入と地域進行

- キリハネは固定個体IDで控えの最初の空きへ1回だけ加入
- 呼称を20文字以内で保存可能
- 先遣隊第1記録と上流弁復旧を同じ地点完了操作で加算
- 研究報告を重複なく2件追加
- 完了後に観測櫓跡を再訪しても報酬を再加算しない
- セーブスキーマを版3へ更新し、版1・版2から移行

## 7. 意図的な制限と次工程

今回の汎用戦闘は、味方3体、敵1体、控え交代なしに限定している。複数敵用の対象候補、交代、状態の残り時間一覧、経験値は未実装である。

次は「沈み水路」を実装し、現在の戦闘モデルへ次を追加する。

1. 敵2体以上と対象選択
2. 汚染状態の継続被害と澄み流しによる解除
3. 環境塊を先に処理する分岐
4. 下流弁復旧と先遣隊第2記録
5. 必要であれば前衛・控え交代

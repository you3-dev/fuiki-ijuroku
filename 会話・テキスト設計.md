# 『封域異獣録』会話・テキスト設計

版：0.1  
更新日：2026年7月13日  
対象：縦切り版

## 1. 採用方針

ゲーム進行中の会話は、通常イベント1回につき2〜5発言程度を基本とする。詳しい世界設定、生態説明、先遣隊の推論は、任意で開ける観察記録と異獣録へ分離する。

```text
今すぐ必要なこと
  → 短い会話、目的表示、操作案内

知ると理解が深まること
  → 観察記録、異獣録、先遣隊記録
```

物語を読まないと操作できない構成にも、説明をすべて飛ばすと世界観が分からない構成にもしない。

## 2. 文章の種類

| 種類 | 役割 | 表示方法 |
|---|---|---|
| イベント会話 | 人物の反応、場面転換 | 画面下部の会話欄 |
| 主人公の観察 | 目の前の事実と仮説 | 記録帳風の短い観察札 |
| 操作案内 | 次にできる操作 | 操作部の近くに短く表示 |
| 現在目的 | 次に達成すること | 研究所と探索画面に固定表示 |
| 観察記録 | 生態、環境、推論 | 任意で展開する記録パネル |
| 異獣録 | 種ごとの蓄積情報 | 異獣録の詳細ページ |
| 先遣隊記録 | 物語の謎と師匠の痕跡 | 独立した復元記録 |
| システム通知 | 保存、入手、解放 | 短い通知。物語口調にしない |

同じ情報を会話、観察記録、異獣録へ長文で重複させない。会話では気付き、記録では根拠、異獣録では確定情報を扱う。

## 3. 文章量の目安

| 場面 | 目安 |
|---|---|
| 通常イベント | 2〜5発言 |
| 操作チュートリアル | 説明1つ＋短い結果1つ |
| 初めての異獣観察 | 3〜6発言＋任意記録 |
| 仲間化成立 | 3〜5発言 |
| 地域分岐の提示 | 2〜4発言 |
| 先遣隊記録 | 150〜300字程度。複数段落へ分ける |
| ボス前後 | 6〜10発言。途中で操作を挟む |
| 観察記録 | 80〜180字程度 |
| 技能・状態説明 | 1〜3文 |

文字数は上限ではなく、iPhoneで一度に読む負担を管理する初期基準とする。

## 4. 1発言の作り方

- 1画面に1つの要点だけを書く
- 1文を短くし、必要なら2文へ分ける
- 固有名詞を一度に複数導入しない
- 操作ボタンの名称と説明文の用語を一致させる
- 感情だけの相づちを連続させない
- 既知の内容を別人物が言い直す会話を避ける
- 暗い雰囲気でも、抽象語だけで状況を隠さない

例として、「遺跡波動の影響により濾過器官が機能不全を起こしている可能性があります」ではなく、「濾過膜の内側に黒い沈殿がある。遺跡波動を取り込みすぎたらしい」と分ける。

## 5. 主人公の文章

主人公は新人調査員だが、受け身の無言役にはしない。目の前の事実を短く観察し、仮説を立てる役割を持つ。

### 主人公が話す内容

- 観察できた事実
- 師匠の教えを思い出した判断
- 異獣を傷つけずに試したい方法
- 調査員としての疑問
- 重要場面での短い意思表示

### 主人公に長く話させない内容

- プレイヤーがまだ見ていない設定の説明
- 他人の感情の断定
- 長い過去語り
- プレイヤーの価値判断を固定する独白

重要な態度だけ選択肢で表し、どれを選んでもチュートリアルの成否が変わらないようにする。

## 6. 操作案内の原則

### 画面に置く案内

```text
観察すると、警戒の原因を調べられます。
[スミワタリを観察]
```

説明文の中で「観察してください」とだけ命令せず、何が分かる操作なのかを先に伝える。

### 結果の案内

```text
濾過膜の詰まりを記録した。
協力条件が1つ判明した。
```

数値変化と物語上の意味を分ける。必要な数値は状態札やゲージへ表示し、会話欄へ詰め込まない。

### ヒント

同じ段階で2ラウンド進展がない場合にだけ表示する。

```text
黒い濁りは体内に残っている。
主人公の調査道具を確認できそうだ。
```

正解となるボタン名を即座に答えとして表示せず、観察内容から操作を推測できる書き方にする。

## 7. 観察記録

観察記録は主人公の調査帳として書く。

### 構成

1. 観察した事実
2. 暫定的な仮説
3. 次に確かめること

### 例

> スミワタリの濾過膜に黒い沈殿が残っている。こちらを威嚇しながらも、攻撃前には必ず腹面を石へ擦りつけた。敵意より、器官の痛みが行動を乱している可能性が高い。まず沈殿を安全に除去できるか試す。

仮説は断定口調にせず、その後の観察で訂正される余地を残す。

## 8. 異獣録との分担

| 調査段階 | 会話で示す | 異獣録へ残す |
|---|---|---|
| 痕跡 | 主人公の違和感 | 目撃地点、仮分類 |
| 初観察 | その場の行動 | 外見、生息環境、暫定生態 |
| 条件発見 | 協力できそうな変化 | 条件の一部または全部 |
| 仲間化 | 個体の反応 | 確定した協力条件、技能候補 |
| 詳細研究 | 新しい疑問 | 系譜、古代技術との関連 |

会話を読み返さなくても、ゲーム上必要な情報は異獣録で確認できるようにする。

## 9. 先遣隊記録

先遣隊記録は長い日記ではなく、調査記録の断片として書く。

### 1記録の構成

- 日付または調査番号
- 観察対象
- 事実
- 師匠の短い判断
- 意図的に残された未解決点

### 文体

- 感情を直接説明しすぎない
- 消去跡、欠損、追記から緊張を示す
- 兵器化計画の名称を初回から明かさない
- 師匠が主人公へ向けた手紙にはしない
- 後から読み直すと意味が変わる情報を1つ含める

記録取得時には要点だけを2〜3行表示し、全文は研究所または探索中の安全な場面で読めるようにする。

## 10. 導入部の表示例

### 10.1 研究所から出発

```text
現在の調査：灰苔湿原
停止した上下流弁と、異獣の異常行動を確認する。
```

主人公の短い観察：

> 先遣隊の記録は、湿原入口で途切れている。まずは同じ地点から痕跡を追う。

操作案内：

```text
トモシゴケとヌマクグリの2体で出発します。
前衛の空き枠には、調査中に協力関係を結んだ異獣を加えられます。
```

### 10.2 湿原入口

```text
主人公：水面だけが揺れている。風ではない。

トモシゴケの苔が、薄い灰色へ変わった。

主人公：先に生物の反応を見る。師匠なら、そうしたはずだ。
```

ここで「周囲を観察」を有効にする。操作可能になる前に説明を続けない。

### 10.3 スミワタリとの遭遇

```text
黒い水が跳ね、半透明の異獣が浅瀬へ滑り出た。

主人公：威嚇している。でも、こちらを見る前から苦しんでいた。

観察すると、行動の原因を調べられます。
```

### 10.4 観察後

```text
濾過膜の内側に、黒い沈殿が詰まっている。

主人公：襲いたいんじゃない。取り込んだものを排出できないんだ。

協力条件：汚染を取り除く　を記録した。
```

### 10.5 鎮静後

```text
トモシゴケの明滅に合わせ、スミワタリの呼吸がゆっくりになる。

警戒度が十分に下がった。
協力を求められます。
```

### 10.6 協力成立

```text
スミワタリは少し離れた水面を一周し、主人公の足元へ戻ってきた。

主人公：一緒に来るなら、濾過膜の状態を継続して記録できる。

スミワタリが調査隊へ加わった。
```

加入後すぐに長い説明を挟まず、呼称入力と前衛3枠の表示へ移る。

## 11. 会話選択肢

縦切り版では、物語を大きく分岐させる会話選択肢を入れない。選択肢を使う場合は、主人公の態度を表す場面に限定する。

例：

```text
研究所へ戻ったら、最初に何を報告する？

[異獣の苦痛について]
[浄化施設の異常について]
[先遣隊の痕跡について]
```

選択によって返答や研究所ホームの報告順は変えても、重要情報、報酬、仲間化条件を失わせない。

## 12. 表示と操作

- タップで次の発言へ進む
- 画面全体の連打ではなく、会話欄または進行ボタンを操作対象にする
- 既読会話のログを現在イベント中に確認できる
- 一度読んだイベントは再調査時に短縮または省略できる
- 初見イベントを誤操作で丸ごと飛ばさない
- 時間制限付きの会話選択を使わない
- 自動送りと音声同期は縦切り版の必須機能にしない
- 読み上げ時に話者名と本文の順序が分かる構造にする

## 13. スキップ規則

| 内容 | 初見 | 既読 |
|---|---|---|
| 通常会話 | 1発言ずつ進める | イベント単位で省略可能 |
| 操作チュートリアル | 省略不可 | 要点確認後に省略可能 |
| 観察記録 | 任意閲覧 | いつでも閲覧可能 |
| 先遣隊記録 | 要点を表示 | 全文を任意閲覧 |
| ボス段階説明 | 短い要点を表示 | 短縮可能 |

スキップしても、現在目的と必要な操作は画面上へ残す。

## 14. データ設計

文章をReactコンポーネントへ直接書かず、イベントデータとして分離する。

```ts
type EventLine = {
  id: string;
  kind: 'dialogue' | 'observation' | 'tutorial' | 'system';
  speakerId?: string;
  text: string;
  portraitId?: string;
};

type TextEvent = {
  id: string;
  trigger: EventTrigger;
  lines: EventLine[];
  repeatPolicy: 'once' | 'shortened' | 'always';
  completionFlag?: string;
};
```

### ID例

```text
intro.laboratory.first_departure
marsh.entrance.first_observation
marsh.shallows.sumiwatari_appears
marsh.shallows.sumiwatari_observed
marsh.shallows.sumiwatari_recruited
```

イベントIDと文章表示済みフラグを分ける。表示途中でアプリが閉じても、報酬や仲間化処理を会話送りによって二重実行しない。

## 15. 文体辞書

用語表記を統一する。

| 使用する語 | 避ける揺れ |
|---|---|
| 異獣 | 魔物、モンスターを説明文で混在させる |
| 調査隊 | パーティー、チームを物語文で使用する |
| 前衛／控え | スタメン／ベンチ |
| 協力要請 | 捕獲、スカウト |
| 共鳴継承 | 配合、合成 |
| 活性値 | MP、マナ |
| 警戒度 | 捕獲率、好感ゲージ |
| 呼称 | ニックネーム。ただし設定画面では説明を添える |

開発用の内部名とプレイヤー向け用語を混同しない。

## 16. テスト項目

- 通常イベントが操作を長時間止めない
- 初回スミワタリ加入まで、1度に教える操作が1つに絞られている
- 会話を読み飛ばしても現在目的を確認できる
- 観察記録から次の行動を推測できる
- 既読イベントを再調査時に短縮できる
- 会話ログで話者と本文を区別できる
- 文字拡大時も発言全文へスクロールして到達できる
- イベント再読込で報酬や個体が二重生成されない
- 異獣録にゲーム上必要な既知条件が残る
- 用語表記が文書と画面で統一されている

## 17. 次に決めること

主人公の発言方法を決める。事実観察は自動で短く発言し、価値判断や対人応答だけを選択肢にする「半固定型」を推奨候補とする。

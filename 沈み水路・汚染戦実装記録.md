# 沈み水路・汚染戦実装記録

## 1. 実装の目的

灰苔湿原の第2分岐「沈み水路」を、地点選択＋短い分岐調査の形式で最後まで遊べるようにした。観測櫓跡の単体戦とは異なり、個体を倒すのではなく、複数対象の汚染を解除して安全な流路を確保する戦闘である。

今回の到達点は次のとおり。

- 取水桝を観察する慎重ルートと、下流弁へ急ぐ強行ルート
- 汚染塊と汚染された野生スミワタリを同時に扱う複数対象戦
- HP、活性値、汚染値、警戒度、鎮静、主人公支援の組み合わせ
- 先遣隊第2記録の回収と下流弁の洗浄
- 観測櫓跡と沈み水路をどちらから攻略しても正しく加算される地域進捗
- 戦闘途中の保存復帰と旧セーブの移行

## 2. 調査分岐

沈み水路へ入ると、戦闘前にアプローチを選ぶ。

| 選択 | 初期状態 | 特徴 |
|---|---|---|
| 取水桝の濁りを観察する（推奨） | 警戒度40、汚染塊60 | 汚染源を特定し、汚染塊への最初の「澄み流し」が60低下になる |
| 下流弁へ急ぐ | 警戒度60、汚染塊80、トモシゴケの汚染1 | 事前観察を省く代わりに、鎮静と浄化へ追加の手数が必要になる |

両方とも同じ報酬へ到達できる。推奨ルートは仕組みを理解しやすく、強行ルートは準備不足が戦闘状態へ直接反映される比較用の選択肢とした。

## 3. 複数対象・汚染戦

### 3.1 対象

- 汚染塊：HPを削らず、汚染値を0にする環境対象
- 汚染された野生スミワタリ：生態HPを維持したまま、汚染値を0にする個体対象
- 前衛3体：トモシゴケ、ヌマクグリ、スミワタリ。敵行動によりHPや汚染状態が変化する

### 3.2 成功条件

次の条件を同時に満たした後、主人公支援「安全な流路を示す」を選ぶ。

1. 汚染塊の汚染値が0
2. 野生スミワタリの汚染値が0
3. 警戒度が20以下
4. 静かな明滅による鎮静が成立済み

協力要請や仲間化は行わない。今回は環境復旧と調査記録の確保が目的である。

### 3.3 失敗と再試行

- 汚染対象を通常攻撃でHP0にした場合は「生態系を損傷」として失敗
- 前衛が全滅した場合も失敗
- 再試行時は戦闘開始時点へ戻し、選んだ慎重／強行アプローチは維持

これにより、討伐ではなく浄化を選ぶことをルールとして明示した。

## 4. ラウンド保存

観測櫓跡と同じく、ラウンド処理を2段階へ分けている。

1. `commitWaterwayRound` が前衛3体の予定行動、対象、主人公支援、乱数状態を確定する
2. 確定状態の保存成功後に `resolveWaterwayRound` が行動を一度だけ解決する

保存中は次の確定操作を無効化する。アプリが2段階の間で終了した場合は、保存済みの確定状態から同じラウンドを再開できる。解決済み状態へ再度解決コマンドを送っても変化しない。

## 5. 地域進捗

戦闘成功直後は結果画面で先遣隊第2記録を表示し、その後に「澄み流しで弁内部を洗浄する」を明示的に選ぶ。洗浄完了時だけ次を更新する。

- 先遣隊記録を1件加算
- 水路弁復旧を1件加算
- 研究報告「下流弁を復旧」を追加
- 沈み水路の完了フラグを設定

完了画面から同じ操作を繰り返しても再加算されない。観測櫓跡と沈み水路の完了フラグから表示値を組み立てるため、どちらを先に攻略しても `1/3・1/2`、両方なら `2/3・2/2` になる。

## 6. 保存互換性

セーブスキーマを版4へ更新した。版1・版2・版3のセーブは読み込み時に次の沈み水路用状態を補完する。

- 選択したアプローチ
- 汚染戦の状態
- 沈み水路の完了フラグ

旧版で沈み水路を選択した直後の状態は、アプローチ選択画面へ移行する。バックアップ読込は版1から版4までを受け付け、検査後に最新版へ変換する。

## 8. 検証結果

| 確認項目 | 結果 |
|---|---|
| 慎重ルートの成功 | 単体テストとブラウザ操作で確認 |
| 強行ルートの成功 | 単体テストで確認 |
| 通常攻撃による生態系損傷 | 失敗判定と再試行を単体テストで確認 |
| 確定済みラウンドの二重解決防止 | 単体テストで確認 |
| 上流→下流の進捗 | 統合テストで `2/3・2/2` を確認 |
| 下流を先に完了した進捗 | 390×844のブラウザ操作で `1/3・1/2` を確認 |
| 版3バックアップの版4移行 | バックアップ単体テストで確認 |
| Lint、型検査、Vitest、本番ビルド | 成功 |

ブラウザでは、推奨アプローチ選択、汚染塊の浄化、野生個体の浄化、鎮静、安全流路の指示、第2記録回収、弁洗浄、研究所帰還までをiPhone相当幅で通した。

## 9. 後続工程

上流弁と下流弁の両方が復旧した後は、正式仕様どおり濾過樹群で遺跡波動を停止し、浄化施設中枢へ接続した。詳細は [濾過樹群・中枢解放実装記録.md](./濾過樹群・中枢解放実装記録.md) を参照する。次は今回の複数対象処理を再利用し、環境装置とニゴリグイの状態が相互に影響する3段階ボス戦を実装する。

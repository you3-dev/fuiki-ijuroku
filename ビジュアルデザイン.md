# 『封域異獣録』ビジュアルデザイン方針

版：0.1  
更新日：2026年7月13日  
対象：縦切り版／iPhone向けPWA

## 1. デザインコンセプト

通常UIは「湿気を含んだ生態調査記録帳」を基調とする。研究所の計測器や整理札を組み合わせ、古代文明の意匠は遺跡装置、共鳴継承、ボス戦など異常性を示す場面に限定する。

```text
通常の調査
  紙、インク、整理札、押印、手描きの地図

研究所
  計測目盛、標本番号、金属留め具、観察窓

古代文明の介入
  青緑の発光線、同心円、欠けた幾何学文様、石質の暗色
```

古代意匠を常時表示しないことで、プレイヤーが遺跡深部へ進んだときの違和感と重要性を強める。

## 2. 視覚上の原則

- 可読性を最優先し、装飾の上へ長文を直接置かない
- 通常画面は暖かい紙色、探索と戦闘は地域背景に合わせて暗さを増す
- 操作部品の形と役割は、通常画面と古代装置画面でも変えない
- 生態情報は図版、見出し、短い観察文、分類札の順に読む構成にする
- 状態は色だけでなく、アイコン、名称、枠線、模様を併用する
- ドラゴンクエストシリーズ固有の書体、紋章、画面枠、効果音、モンスター造形を模倣しない

## 3. 基本配色

### 通常UI

| 役割 | 色 | 用途 |
|---|---|---|
| 紙地 | `#E7E0CC` | アプリ全体の基本背景 |
| 明るい紙 | `#F4EFDF` | カード、説明欄、ダイアログ |
| 濃いインク | `#232923` | 本文、主要見出し |
| 薄いインク | `#596158` | 補足、未選択情報 |
| 苔緑 | `#506B55` | 主要操作、調査済み、回復 |
| 深い苔緑 | `#344A3A` | 押下状態、強調枠 |
| 水路青 | `#527477` | 湿潤、浄化、環境情報 |
| 錆赤 | `#95584B` | 危険、保存失敗、汚染警告 |
| 黄土 | `#A77E3F` | 新情報、研究成果、注意 |
| 境界線 | `#AAA28D` | カード境界、目盛、区切り |

### 古代文明の差し色

| 役割 | 色 | 用途 |
|---|---|---|
| 古代発光 | `#70C0B4` | 生命核、共鳴線、起動中の装置 |
| 古代深光 | `#287A76` | 発光部の中心、確定済み共鳴 |
| 石質暗色 | `#252D30` | 遺跡装置、ボス戦の暗い盤面 |
| 過負荷 | `#C36A4D` | 装置の危険状態、過負荷計 |

古代発光色を通常の主要ボタンには使用しない。「古代技術が動いている」という物語上の意味を保つ。

### 配色の検証

上記は初期候補であり、実装時に本文、ボタン、状態表示のコントラストを実機で検証する。基準を満たさない組合せは、色相を守ることより明暗差を優先して調整する。

## 4. 文字

### 基本方針

- 本文は端末標準の日本語ゴシック系を使う
- Webフォントを必須にせず、初回読込みとオフライン動作を軽くする
- 長い観察記録にも耐える行間を確保する
- 数値、技能名、状態名を小さく詰め込みすぎない
- 装飾書体の代わりに、罫線、押印、字間、背景札で世界観を出す

### 文字の階層

| 階層 | 用途 | 表現 |
|---|---|---|
| 画面見出し | 研究所、異獣録など | 太め、短く、上部に固定 |
| 節見出し | 現在の調査、観察記録 | 太め、左側に分類線 |
| 本文 | 説明、物語、記録 | 標準ウェイト、十分な行間 |
| ラベル | HP、活性値、警戒度 | 短い語＋数値、背景札 |
| 注記 | 未調査条件、補足 | 本文より弱くするが読める濃さ |

古代装置の文字を未知の記号だけにしない。操作に必要な部分には必ず日本語の名称と説明を併記する。

## 5. 形と質感

### 通常カード

- 角をわずかに丸めた紙片
- 細い境界線と控えめな影
- 上部に標本番号または分類札
- 汚れや紙目は背景の低いコントラストに限定

### 研究所の部品

- 金属留め具
- 計測目盛
- 小さな状態灯
- 標本瓶を思わせる縦長の状態枠

装飾はCSSまたは小さな共通画像で再利用し、画面ごとに別の豪華な枠を描かない。

### 古代装置

- 暗い石または金属の面
- 欠損した同心円
- 細い発光線
- 生物の脈動を思わせるゆっくりした明滅

古代装置でも主要操作は通常ボタンと同じ位置・形を保つ。発光線そのものを押させるような分かりにくい操作は避ける。

## 6. UI部品

### ボタン

| 種類 | 用途 | 表現 |
|---|---|---|
| 主要 | 調査開始、ラウンド確定 | 苔緑の面、明るい文字 |
| 副次 | 詳細、並べ替え、戻る | 明るい紙、濃い枠 |
| 控えめ | 説明、絞込み | 文字＋小さなアイコン |
| 危険 | 中断、削除、破棄 | 錆赤。通常操作から距離を置く |
| 古代確定 | 共鳴継承の最終実行 | 古代発光の枠＋明確な日本語 |

重要操作は画面内に複数並べない。「ラウンド確定」など現在の主操作を1つ明確にする。

### 状態札

湿潤、霧中、汚染、装甲、露出、鎮静は、小さな分類札として表示する。

- 固有アイコン
- 状態名
- 残りラウンド
- 必要時のみ短い効果説明

例：汚染は斑点模様、装甲は重なった板、露出は割れ目の記号を併用する。

### ゲージ

- HP：生命状態として苔緑から錆赤へ変化
- 活性値：水路青または淡い青緑
- 警戒度：黄土から錆赤
- ボス過負荷：古代発光から過負荷色

色付きの棒だけにせず、現在値、最大値、名称を表示する。大きな変化時だけ短い増減演出を加える。

## 7. 生態図鑑表現

異獣の立ち絵は「戦闘用キャラクター画像」だけでなく、研究記録の図版として見せる。

```text
┌─────────────────────┐
│ 標本記録 GM-04          │
│                         │
│       異獣の全身図        │
│      薄い生息環境線画      │
│                         │
├─────────────────────┤
│ キリハネ                 │
│ 調査段階：生態確認         │
│ 生息：灰苔湿原・観測櫓跡   │
│ 観察記録：……              │
└─────────────────────┘
```

### 調査段階による変化

| 段階 | 図版と情報 |
|---|---|
| 未発見 | 枠自体を表示しない、または地域の痕跡だけ表示 |
| 痕跡確認 | 不鮮明な輪郭、仮分類、目撃地点 |
| 初観察 | 全身図、暫定名、短い行動記録 |
| 生態確認 | 技能、協力条件、生息環境 |
| 詳細研究 | 系譜、古代技術との関連、追加記録 |

情報が増えるたびに図版へ書込みや分類札が追加され、「ページが完成していく」感覚を作る。

## 8. 画面ごとの濃度

| 画面 | 記録帳 | 研究機器 | 古代意匠 |
|---|---:|---:|---:|
| 研究所 | 高 | 中 | 低 |
| 編成 | 中 | 高 | 低 |
| 異獣録 | 最高 | 低 | 低〜中 |
| 灰苔湿原の探索 | 中 | 低 | 地域奥ほど増加 |
| 通常戦闘 | 低 | 中 | 敵や地点に応じる |
| 共鳴継承 | 中 | 中 | 高 |
| ニゴリグイ戦 | 低 | 中 | 最高 |

この濃度差により、同じUI部品を使いながら物語上の深度を示す。

## 9. 灰苔湿原への適用

### 地域色

- 灰緑の苔
- 濁った青灰色の水
- 湿った紙の褐色
- 装置の青緑発光
- 汚染の錆赤と黒い斑点

### 地点による変化

| 地点 | 視覚の特徴 |
|---|---|
| 湿原入口 | 紙色と自然色が中心。古代発光はほぼない |
| 灰苔の浅瀬 | 水路青と苔緑。小さな異常記号が現れる |
| 観測櫓跡 | 研究目盛と古い計測器を強める |
| 沈み水路 | 暗い水面、汚染斑点、欠けた発光線 |
| 濾過樹群 | 自然の枝と人工的な同心円が混ざる |
| 浄化施設中枢 | 石質暗色と古代発光が画面を支配する |

## 10. 戦闘の視認性

- 背景を暗くまたはぼかし、立ち絵と状態表示を前へ出す
- 味方と敵の情報枠で形を変える
- 選択中の味方は輪郭、名前札、位置の3点で示す
- 技能の対象候補を明るくし、対象外を弱める
- 環境状態は戦場上部の固定位置へまとめる
- ダメージ数値だけに頼らず、HPゲージの変化を残す
- 状態付与時はアイコンと名称を短く表示する

派手な全画面演出より、3体分の行動結果を追えることを優先する。

## 11. 動き

縦切り版では共通の短い動きだけを用意する。

- 待機：わずかな上下または呼吸
- 攻撃：短い前進と反動
- 被弾：小さな揺れと明度変化
- 回復：苔色の粒子が上へ流れる
- 浄化：水路青の波紋
- 鎮静：動きがゆっくり収束する
- 古代起動：線が順に接続して発光する

常に動き続ける背景や強い点滅を避ける。端末の動きを減らす設定に応じて、移動量と反復を減らせるようにする。

## 12. アイコン

- 線幅と角の処理を統一する
- 24px前後でも意味を判別できる単純な形にする
- アイコン単独で重要操作を表さず、原則として名称を併記する
- 異獣の属性表ではなく、生態状態と調査行動を中心に作る
- 既存ゲームの呪文、職業、モンスターを連想させる固有記号を避ける

縦切り版で優先するアイコンは、研究所、編成、異獣録、観察、警戒、協力、湿潤、霧中、汚染、装甲、露出、鎮静、活性値、共鳴、保存、オフラインである。

## 13. 素材形式

| 素材 | 基本形式 | 備考 |
|---|---|---|
| 異獣立ち絵 | 透過WebPまたはPNG | 図鑑と戦闘で共用 |
| 背景 | WebP | 地点ごとに静止画1枚を基本 |
| UIアイコン | SVG | 色変更と小型表示に対応 |
| 発光マスク | 透過画像またはSVG | 古代装置、生命核の共通演出 |
| 紙目・汚れ | 小さな反復画像 | 低コントラスト、使い回す |

制作原寸とゲーム内表示用を書き分け、PWAには表示に必要な大きさへ圧縮した素材だけを含める。

## 14. デザイントークン候補

実装では色や余白を直接散在させず、CSSカスタムプロパティへ集約する。

```css
:root {
  --color-paper: #e7e0cc;
  --color-paper-raised: #f4efdf;
  --color-ink: #232923;
  --color-ink-muted: #596158;
  --color-moss: #506b55;
  --color-water: #527477;
  --color-rust: #95584b;
  --color-ochre: #a77e3f;
  --color-border: #aaa28d;
  --color-ancient-glow: #70c0b4;
  --color-ancient-stone: #252d30;

  --radius-card: 0.75rem;
  --radius-control: 0.625rem;
  --space-1: 0.25rem;
  --space-2: 0.5rem;
  --space-3: 0.75rem;
  --space-4: 1rem;
  --space-6: 1.5rem;
}
```

数値は実機モックを見て調整するが、名前と用途は維持する。

## 15. 縦切り版で作る視覚素材

- 6種の全身立ち絵
- 6種の一覧用アイコン
- 必要な発光マスク
- 灰苔湿原の地点背景6枚
- 研究所背景またはホーム用部分背景1枚
- 生態状態と主要操作のアイコン
- 紙目、分類札、計測目盛の共通素材
- PWAアイコンと起動時表示用素材

戦闘専用の別立ち絵、個体ごとの連続アニメーション、全技能固有の大規模エフェクトは作らない。

## 16. テスト項目

- 紙色の上で本文と補足を区別して読める
- 屋外や画面輝度を下げた状態でも主要操作を判別できる
- 状態をモノクロ表示でも名称と形で区別できる
- 古代発光が通常の主要ボタンと混同されない
- 異獣録の調査段階が図版と文章の両方で分かる
- 灰苔湿原の奥へ進むほど古代技術の影響が増したと感じられる
- 戦闘背景の上でHP、活性値、行動予定を追える
- 動きを減らす設定でも進行と結果を理解できる
- オフライン時にWebフォント欠落でレイアウトが崩れない

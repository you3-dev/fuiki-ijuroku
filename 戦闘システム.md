# 『封域異獣録』戦闘システム

版：0.1  
更新日：2026年7月13日

## 1. 設計目標

- 最大3体の前衛の役割と組み合わせを考える楽しさを作る
- 1ラウンドの指示をiPhone縦画面で無理なく完了できる
- 強い技能の連打ではなく、基礎行動、防御、技能、交代を使い分ける
- 観察と鎮静を、通常戦闘と同じルール上で扱う
- 通常戦を1〜3分、ボス戦を5〜10分に収める
- 計算結果を自動テストで再現できる

## 2. 編成

- 調査隊：最大6体
- 前衛：1〜3体。通常は3体編成を推奨する
- 控え：最大3体
- 敵：通常1〜3体
- 大型ボス：1体＋環境装置または追加行動

## 3. ラウンド

1. ラウンド開始時効果を処理する。
2. 主人公の調査支援を選ぶ。
3. 現在行動可能な前衛1〜3体の行動と対象を選ぶ。
4. 予定行動を確認・修正する。
5. 「行動開始」でラウンドを確定保存する。
6. 行動優先度と速度に基づいて実行順を決める。
7. 行動を順番に処理する。
8. 継続効果、状態異常、環境効果を処理する。
9. 勝利、敗北、協力成立を判定する。

## 4. コマンド

### 技能

装備している最大4個の技能から選ぶ。活性値を消費する。

縦切り版の8技能と数値は [技能・状態設定.md](./技能・状態設定.md) を参照する。

### 基礎行動

個体ごとに設定された通常攻撃または基本支援。活性値を回復する。

### 防御

そのラウンドの被害を軽減し、活性値を基礎行動より多く回復する。

### 交代

控えと入れ替わる。任意交代はその個体の行動を消費し、交代先は同じラウンドに通常行動できない。

### 道具

回復用品ではなく、観察、鎮静、環境対処を中心にする。使用者の行動を消費するか、主人公支援枠を使うかは道具ごとに定義する。

## 5. 活性値

### 基本

- 範囲：0〜100
- 戦闘開始時：40
- 戦闘終了後：次戦闘の初期値へ戻る
- 最大値はMVPでは全個体100で固定する

### 増減の初期案

| 行動 | 活性値 |
|---|---:|
| 基礎行動 | +20 |
| 防御 | +25 |
| 一般技能 | -20〜30 |
| 強力な技能 | -40〜60 |

特性による増減は、基本ルールのプレイテスト後に追加する。

### 表示

- HPの下に細い活性ゲージを表示する
- 使用可能な技能は通常表示する
- 不足する技能は暗くし、必要値と不足値を表示する
- 活性値が増減する行動には、実行前の予測値を表示する

## 6. 能力値

| 能力 | 用途 |
|---|---|
| 生命 | 最大HP |
| 攻勢 | 噛みつき、体当たり、爪などの物理技能の威力 |
| 堅牢 | 物理技能による被害の軽減 |
| 感応 | 生命核、霧、浄化、回復などの生態技能の威力 |
| 抵抗 | 生態技能による被害と状態異常への耐性 |
| 機敏 | ラウンド内の行動順 |

### 設計ルール

- 活性値は能力値に含めない
- 命中と回避を常設能力にしない
- 機敏は原則として行動順だけに使う
- 物理技能は攻勢と堅牢を参照する
- 生態技能は感応と抵抗を参照する
- 回復技能は使用者の感応を参照する
- 固定効果の観察・環境操作は能力値を参照しない場合がある

### 縦切り版6種の傾向

| 種 | 高い能力 | 低い能力 |
|---|---|---|
| トモシゴケ | 感応 | 攻勢、堅牢 |
| ヌマクグリ | 生命、堅牢 | 機敏 |
| スミワタリ | 感応、抵抗 | 攻勢 |
| キリハネ | 機敏、感応 | 生命、堅牢 |
| レキマトイ | 攻勢、堅牢 | 感応、機敏 |
| ニゴリグイ | 生命、抵抗 | 機敏 |

## 7. ダメージと回復

### 物理ダメージ

```text
floor(
  技能威力
  × (50 ＋ 使用者の攻勢) ÷ (50 ＋ 対象の堅牢)
  × 乱数補正
)
```

### 生態ダメージ

```text
floor(
  技能威力
  × (50 ＋ 使用者の感応) ÷ (50 ＋ 対象の抵抗)
  × 乱数補正
)
```

### 回復

```text
floor(
  技能回復力
  × (1 ＋ 使用者の感応 ÷ 100)
)
```

### 共通ルール

- 乱数補正：0.95〜1.05
- ダメージの最低値：1
- 小数点以下：最後に切り捨て
- 防御中：最終ダメージを60%にする
- 回復：最大HPを超えない
- 会心攻撃：MVPでは実装しない
- 乱数：戦闘状態へ保存可能なシードから生成する

### 予測表示

行動確定前に、現在判明している能力と効果を使って予想ダメージまたは回復範囲を表示できる設計にする。未知の敵能力を完全に開示するかは図鑑段階によって制御する。

## 8. 属性と生態状態

### 固定属性表

縦切り版では採用しない。植生、湿生、翅虫、鉱殻などの系統タグは、図鑑、共鳴継承、イベント条件へ使い、単純な弱点倍率へ直結させない。

### 生態状態

| 状態 | 対象 | 基本効果 |
|---|---|---|
| 湿潤 | 個体または戦場 | 霧・泥技能の条件になり、一部効果が強まる |
| 霧中 | 戦場 | 視界を妨げ、キリハネ等の技能条件になる |
| 汚染 | 個体 | ラウンド終了時の被害または能力低下。浄化で解除できる |
| 装甲 | 個体 | 物理被害を軽減する |
| 露出 | 個体 | 装甲解除後に付与され、物理被害が増える |
| 鎮静 | 野生個体 | 攻勢と警戒度を下げ、協力要請へ近づける |

### 共通ルール

- 同名状態は原則として重複しない
- 再付与時は持続時間を更新する
- 状態の残りラウンド数をアイコン上へ表示する
- 状態を色だけで区別しない
- 戦場状態と個体状態を分けて表示する
- 状態相互作用は技能説明と行動予測へ明記する

### 付与と持続時間

技能の対象・条件を満たした状態変化は原則として成功する。確率抽選は行わない。

```text
持続時間
＝ clamp(
    基本2 ＋ floor((使用者の感応 − 対象の抵抗) ÷ 20),
    1,
    3
  )
```

- 能力差が小さい場合は2ラウンド
- 使用者の感応が20以上高い場合は3ラウンド
- 対象の抵抗が20以上高い場合は1ラウンド
- 環境装置等による状態はデータで持続時間を固定できる
- 完全耐性は能力値ではなく明示的な耐性タグで定義する
- ボスの完全耐性を多用せず、短時間でも効果が通る設計を優先する
- 完全耐性または短縮結果を行動確定前に表示する

## 9. 入力短縮

- 前ラウンドと同じ
- 前衛全員へ基礎行動を設定
- 対象候補が1体なら対象選択を省略
- 予定行動一覧から任意の個体へ戻る
- 演出速度：標準、高速、最速

## 10. 主人公の調査支援

味方異獣の行動とは別枠で、原則として1ラウンドに1回使用できる。

- 観察
- 協力要請
- 調査道具
- 環境装置への操作

主人公自身はHPを持つ戦闘ユニットとして扱わない。敵の攻撃対象にもならない。

## 11. 勝敗と終了

- 敵対個体がすべて戦闘不能：勝利
- 出撃中の前衛と控えがすべて戦闘不能：敗北
- 協力要請等によって敵対個体がいなくなる：調査成功
- ボス固有の鎮静条件を満たす：ボス調査成功

## 12. 調査失敗と敗北

ニゴリグイ戦で味方が全滅するか本体HPをゼロにした場合は、ボス戦直前のチェックポイントへ戻る。研究資源、図鑑、地域解放状態は失わない。

通常調査の敗北時処理は別途決定する。

# 『封域異獣録』画面構成設計

版：0.1  
更新日：2026年7月13日  
対象：縦切り版／iPhone縦画面

## 1. 採用方針

通常時は画面下部に「研究所」「編成」「異獣録」の3タブを固定する。共鳴継承と設定は研究所から開く。探索と戦闘では下部タブを隠し、調査専用の全画面表示へ切り替える。

```text
通常シェル
┌─────────────────────┐
│ 画面名            状態 │  上部バー
├─────────────────────┤
│                     │
│   画面ごとの内容      │  スクロール領域
│                     │
├─────────────────────┤
│ 研究所 │ 編成 │ 異獣録 │  下部3タブ
└─────────────────────┘

集中シェル（探索・戦闘）
┌─────────────────────┐
│ 戻る／地域名      状態 │
├─────────────────────┤
│                     │
│  探索または戦闘内容   │
│                     │
├─────────────────────┤
│ その場で必要な操作     │
└─────────────────────┘
```

## 2. 画面シェル

### 2.1 通常シェル

研究所、編成、異獣録で共通使用する。

| 領域 | 内容 |
|---|---|
| 上部バー | 画面名、保存状態、必要時のみ補助操作 |
| 本文 | 画面ごとのカード、一覧、詳細 |
| 下部タブ | 研究所、編成、異獣録 |

下部タブはiPhoneの安全領域を含む高さを確保する。タブはアイコンと名称を併記し、現在位置を色だけでなく形状や下線でも示す。

### 2.2 集中シェル

探索と戦闘で使用する。

- 下部3タブを表示しない
- 地域名、現在地点、戦闘段階など、その場で必要な情報を優先する
- 研究所へ戻る操作は、誤タップしにくい確認付き操作にする
- 全体設定ではなく、音声や説明など必要最小限の補助操作だけを置く
- アプリ切替後も、確定済み状態から同じ画面へ復帰する

### 2.3 特別画面

主人公設定、起動読込、セーブ復旧は下部タブを持たない。処理の目的を1つに絞り、完了後に研究所へ移る。

## 3. 下部3タブ

| タブ | 主な目的 | 開く画面 |
|---|---|---|
| 研究所 | 次の行動を決める | 研究所ホーム |
| 編成 | 異獣を整える | 前衛・控え、所有異獣、育成確認 |
| 異獣録 | 調査成果を読む | 種一覧、調査段階、詳細記録 |

### 操作規則

- タブ切替だけではゲーム進行やセーブ内容を変更しない
- 選択中のタブを再度押すと、その画面の先頭または基準位置へ戻す
- 未確定の編成変更がある場合、別タブへ移る前に破棄確認を出す
- タブごとの一覧位置は同一セッション中だけ保持してよい
- アプリ再起動後は、保存されたゲーム進行に応じた画面を優先する
- 通知バッジは「新しい図鑑情報」など行動可能な更新に限定する

## 4. 研究所ホーム

研究所はメニュー一覧ではなく、「次に何をすればよいか」が分かるホーム画面にする。

### 情報の優先順位

1. 現在の調査目的
2. 調査を開始・再開する操作
3. 調査隊の状態
4. 新しい研究成果
5. 共鳴継承など研究所内の機能
6. 設定とバックアップ

### 画面骨格

```text
┌─────────────────────┐
│ 研究所          保存済 │
├─────────────────────┤
│ 現在の調査              │
│ 灰苔湿原：上下流弁の調査  │
│ [調査を開始／再開]        │
├─────────────────────┤
│ 調査隊                   │
│ 前衛3体＋控え3体の要約     │
│ [編成を確認]              │
├─────────────────────┤
│ 研究報告                 │
│ 新しい異獣録・記録・触媒    │
├─────────────────────┤
│ [共鳴継承] [設定・保存]     │
├─────────────────────┤
│ 研究所 │ 編成 │ 異獣録     │
└─────────────────────┘
```

設定アイコンだけを上部へ置いて見落とさせず、「設定・保存」という文章付き導線も研究所内へ用意する。

## 5. 編成画面

### 目的

- 前衛3体と控え3体を確認する
- 所有異獣から交代要員を選ぶ
- 能力、技能、信頼度、成長傾向を比較する
- 調査へ出る前に不足役割へ気付く

### 画面骨格

```text
┌─────────────────────┐
│ 編成              確定 │
├─────────────────────┤
│ 前衛                    │
│ [1] [2] [3]             │
│ 控え                    │
│ [4] [5] [6]             │
├─────────────────────┤
│ 選択中の異獣             │
│ 立ち絵／役割／能力要約     │
│ 技能・状態対応            │
├─────────────────────┤
│ 所有異獣一覧              │
│ [カード] [カード] ...     │
├─────────────────────┤
│ 研究所 │ 編成 │ 異獣録     │
└─────────────────────┘
```

ドラッグ操作だけに依存しない。枠を選んで交代相手を選ぶ方式を必ず用意し、タップだけで編成を完了できるようにする。並べ替え中は編集用状態に置き、「確定」で一括保存する。

## 6. 異獣録

### 一覧

- 生態図鑑風の小さな図版
- 名前または未同定表示
- 調査段階
- 新しく判明した情報の印
- 役割や生息地による簡単な絞込み

### 詳細

- 全身図
- 観察記録
- 生息環境
- 行動・技能
- 協力条件
- 古代技術との関連
- 共鳴継承で判明した系譜情報

未調査情報は空欄ではなく、「痕跡未確認」「追加観察が必要」など次の調査行動につながる文章で示す。

```text
┌─────────────────────┐
│ 異獣録          絞り込み │
├─────────────────────┤
│ 調査済 4 / 6             │
│ [図版 名前    調査段階]    │
│ [図版 名前    新情報あり]  │
│ [影絵 未同定   痕跡のみ]    │
│ ...                      │
├─────────────────────┤
│ 研究所 │ 編成 │ 異獣録     │
└─────────────────────┘
```

## 7. 共鳴継承

共鳴継承は研究所から開く独立した大画面とするが、常設タブにはしない。縦切り版では使用頻度が編成や異獣録より低く、重要な確定操作を含むためである。

### 手順

1. 基体を選ぶ
2. 共鳴相手を選ぶ
3. 任意の触媒を選ぶ
4. 幼体候補と継承容量を確認する
5. 特性、技能、成長傾向を選ぶ
6. 最終確認して実行する

画面上部に現在の段階を示す。途中の戻る操作では一段階戻り、画面全体から離れる場合だけプレビュー破棄を確認する。

## 8. 探索画面

地点選択版でも「地図を進んでいる感覚」を残すため、選択肢だけの文章画面にはしない。

```text
┌─────────────────────┐
│ 灰苔湿原       調査中断 │
│ 記録 1/3・弁 0/2          │
├─────────────────────┤
│ 地域図／現在地点           │
│ ●入口 ─ ●浅瀬 ─ ○未調査   │
├─────────────────────┤
│ 灰苔の浅瀬                │
│ 背景・短い状況説明          │
│ [観測櫓跡へ]               │
│ [沈み水路へ]               │
│ [周囲を観察]               │
├─────────────────────┤
│ 調査隊の簡易状態            │
└─────────────────────┘
```

- 地域図、現在地点、未調査地点を同時に示す
- 分岐の結果を選択前に完全には明かさない
- 危険、観察、記録、装置など選択の性質はアイコンと文章で示す
- 戦闘開始時は確定保存後に戦闘画面へ移る
- 調査中断は確認後、仕様で定めた確定地点へ戻す

## 9. 戦闘画面

前衛3体へ個別指示するため、表示と入力を上下に分ける。

```text
┌─────────────────────┐
│ 灰苔湿原       ラウンド3 │
├─────────────────────┤
│ 敵1〜3体／環境状態         │
│       生態図鑑風立ち絵      │
├─────────────────────┤
│ 味方前衛3体の状態           │
│ HP・活性値・状態・行動予定   │
├─────────────────────┤
│ 選択中：トモシゴケ          │
│ [基礎][技能][防御][交代]     │
│ 行動説明／対象選択           │
├─────────────────────┤
│ [前回を再現] [全員基礎]      │
│ [3体分の行動を確認]          │
└─────────────────────┘
```

3体分の入力状況を常に確認できるよう、各前衛に「未入力」「入力済み」を示す。最終確認画面から任意の1体へ戻って修正できるようにする。

## 10. ダイアログと補助表示

次は新しいルートではなく、現在画面の上に重ねる。

- 技能と状態の説明
- アイテム説明
- 未確定変更の破棄確認
- 調査中断の確認
- 戦闘から離れる確認
- 保存失敗と再試行
- PWA更新の適用確認

重要な確定操作は、ボタンの名称を「はい」だけにせず、「共鳴継承を実行」「未確定編成を破棄」のように結果が分かる文章にする。

## 11. レスポンシブ方針

- iPhone縦画面を設計基準とする
- 本文だけをスクロールさせ、重要な確定操作を見失わせない
- 横スクロールを主要操作に使用しない
- 立ち絵が小さくなる端末では、装飾より能力値と操作を優先する
- 画面の高さが小さい場合も、確定ボタンと現在の選択対象を同時に確認できる構成にする
- 横向きやPCでは中央に適切な最大幅を設け、極端に横へ引き伸ばさない

## 12. 画面共通の状態表示

| 状態 | 表示場所 |
|---|---|
| 保存中・保存済み・保存失敗 | 通常画面の上部バー。失敗時は固定通知も表示 |
| オフライン | 上部の小さな状態表示。プレイ可能なら警告色を強くしすぎない |
| 更新待ち | 研究所または設定。戦闘・共鳴中は表示だけにする |
| 新しい異獣録情報 | 異獣録タブと該当種 |
| 調査目的更新 | 研究所の最上位カード |
| 編成不備 | 編成画面と調査開始前の確認 |

## 13. テスト項目

- 小型iPhone相当の縦画面で横スクロールが発生しない
- 下部タブが安全領域やホームインジケータと重ならない
- 3タブを片手操作で切り替えられる
- 現在のタブを色なしでも判別できる
- 研究所を開いて次の目的と調査開始操作を見つけられる
- タップ操作だけで前衛3体＋控え3体を編成できる
- 戦闘中に3体分の入力状態を確認できる
- 探索・戦闘中に下部タブが誤操作されない
- 文字サイズを拡大しても主要な確定操作へ到達できる
- 保存失敗とオフラインを区別できる

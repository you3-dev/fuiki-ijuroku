# 『封域異獣録』登場人物設定

版：0.1  
更新日：2026年7月13日  
対象：縦切り版の研究所常駐人物

## 1. 採用構成

研究所で常時会話する人物は2人に絞る。

| 人物 | 役割 | 物語上の視点 |
|---|---|---|
| 研究主任 リゼ・オルド | 調査判断、安全管理、生態研究 | 生きている異獣と現場の変化を優先する |
| 記録技師 ユノ・ケイル | 古代装置、記録復元、共鳴設備 | 再現可能な証拠と失われた技術を優先する |

失踪した師匠は常駐人物に含めず、先遣隊記録と主人公の短い回想を通して存在感を出す。

名前は制作上の正式候補とする。実装前でも変更できるよう、内部IDと表示名を分離する。

## 2. 研究主任 リゼ・オルド

### 基本情報

| 項目 | 内容 |
|---|---|
| 内部ID | `lize_ordo` |
| 役職 | 辺境研究所・研究主任 |
| 年代 | 40代前半 |
| 専門 | 異獣の群集生態、封鎖地域の安全評価 |
| 主な場所 | 研究所の調査卓 |
| 主人公との関係 | 新人調査員の直属責任者 |

### 人物像

現場経験のある生態研究者。異獣を危険度だけで分類することには反対するが、無条件の保護も主張しない。観察記録と退路を確保したうえで接触することを求める。

先遣隊失踪後、辺境研究所の調査責任を一時的に引き継いだ。主人公を灰苔湿原へ送ることに慎重だが、師匠が残した推薦と主人公自身の観察能力を尊重し、完全には止めない。

### 表向きの目的

- 灰苔湿原の異常が周辺水系へ拡大するのを防ぐ
- 新人を含む調査員を生還させる
- 異獣の排除ではなく、生態系の回復を優先する
- 未確認情報を断定して上部組織へ送らない

### 内面の問題

先遣隊の出発を承認した責任を感じている。危険を避けようとする判断が、結果として真相の隠蔽に加担するのではないかと恐れている。

主人公にはその罪悪感を語らず、「証拠が足りない」という言葉で慎重さを保つ。

### 会話の特徴

- 一文が短く、結論を先に言う
- 感情を命令口調で隠さない
- 主人公の仮説を否定する前に根拠を尋ねる
- 成功を大げさに褒めず、観察の具体的な点を評価する
- 危険時には遠回しにせず帰還を指示する

### 台詞例

```text
記録は読んだ。スミワタリを刺激した原因は、君たちではない。

ただし、次も同じ手順で鎮静できるとは限らない。環境の変化を先に見て。

先遣隊の件は、まだ推測で報告しない。証拠を持ち帰れば、私が止めさせない。
```

### ゲーム上の担当

- 現在目的の提示
- 調査開始と帰還報告
- 編成不足と危険度の警告
- 地域解放条件の説明
- ボス戦前の最終確認
- プレイテスト完了時の調査総括

研究主任の会話を読まないと出発できない構成にはしない。要点は現在目的カードにも表示する。

## 3. 記録技師 ユノ・ケイル

### 基本情報

| 項目 | 内容 |
|---|---|
| 内部ID | `yuno_keil` |
| 役職 | 記録復元・装置技師 |
| 年代 | 20代後半 |
| 専門 | 古代記録媒体、生命核計測、共鳴設備の保守 |
| 主な場所 | 研究所の復元卓と共鳴槽 |
| 主人公との関係 | 調査データを復元する技術担当 |

### 人物像

壊れた記録媒体や古代装置を、推測ではなく再現試験で読み解く技師。異獣の感情を軽視しているように見えるが、生命核を消耗品として扱う既存の実験手法を嫌っている。

非消失型の共鳴継承設備を、安全側へ改修した中心人物の1人。提供元が残ることを技術上の最低条件としている。

### 表向きの目的

- 先遣隊記録を欠損したまま断定しない
- 古代装置の命令系統を再現する
- 生命核を損なわない計測手順を確立する
- 調査データとセーブ上の変化を対応付ける

### 内面の問題

先遣隊から届いた最後の破損信号を、事故直後に一度だけ復元している。しかし、正式な記録へ残す前に上部組織から削除命令が届いた。

ユノは完全な内容を覚えていないと主張するが、復元卓へ断片的な照合値を残している。縦切り版では裏切り者にはせず、証拠の扱いに慎重な協力者とする。

### 会話の特徴

- 観測値と欠損箇所を明確に区別する
- 乾いた短い冗談を時々使う
- 古代技術を神秘化せず、壊れた装置として扱う
- 主人公へ「信じるか」ではなく「再現できるか」を尋ねる
- 異獣の反応を計測値だけで終わらせず、主人公の観察記録と照合する

### 台詞例

```text
記録の六割は泥、二割は熱損傷。残り二割に、消された形跡がある。

古代装置だから難しいんじゃない。説明書を残した人が、全員いないから難しい。

共鳴値は安定。提供元の生命核にも減衰なし。ようやく「成功」と記録できる。
```

### ゲーム上の担当

- 先遣隊記録の復元
- 古代装置の状態説明
- 研究素材と触媒の確認
- 共鳴継承のチュートリアル
- JSONバックアップを世界観へ寄せた「調査記録の書出し」として案内
- セーブ復旧画面の補助説明

保存失敗などの技術的な問題をユノの物語台詞だけで説明しない。システム上の正確なエラー表示を併記する。

## 4. 2人の関係

リゼとユノは対立陣営ではない。同じ目的へ異なる証拠から近づく。

| 論点 | リゼ | ユノ |
|---|---|---|
| 優先する証拠 | 生態の変化、個体の行動 | 記録媒体、装置の応答 |
| 危険時 | 調査員と異獣を退避させる | 装置を停止し記録を保全する |
| 先遣隊 | 生存可能性を捨てない | 残された信号を疑わない |
| 古代技術 | 生態へ与える影響を警戒 | 再現条件と改修可能性を調べる |
| 主人公への期待 | 現場で変化を見つける | 欠損した証拠を持ち帰る |

議論は2〜4発言で終え、主人公の前で同じ説明を繰り返さない。

### 会話例

```text
リゼ：装置を止めれば、水路の異獣がどう動くか分からない。

ユノ：止めなければ、今の命令が続く。それは分かってる。

主人公の選択：
[先に周囲の異獣を観察する]
[停止前後の値を記録する]
```

どちらを選んでも、観察と記録の両方を実行する。違いは先に行う手順と2人の返答だけにする。

## 5. 先遣隊長との関係

### リゼ

- 先遣隊長とは同世代の研究者
- 灰苔湿原の調査計画を承認した
- 帰還期限を過ぎた後も捜索継続を主張した
- 主人公へは、師匠の推薦内容をすべて見せていない

### ユノ

- 先遣隊の記録装置を整備した
- 最後の破損信号を受信した
- 先遣隊長が記録を複数地点へ分散した意図を技術面から理解する
- 記録3個が揃うまで、断片の推測を主人公へ断定しない

師匠を2人の過去話だけで説明せず、主人公が回収する記録の内容を中心に人物像を作る。

## 6. 縦切り版での登場順

### 主人公設定後

リゼが現在の調査目的と帰還条件を伝える。ユノは初期同行2体の生命核計測と記録端末を確認する。

### 初回出発前

- リゼ：灰苔湿原の危険と撤退基準
- ユノ：記録端末と汚染除去具
- 会話全体：5発言以内

### スミワタリ加入後

- リゼ：苦痛の原因を観察した点を評価
- ユノ：汚染除去前後の濾過値を比較
- 主人公の選択に応じて報告順を変える

### 第1記録回収後

リゼは先遣隊の行動が合理的だったと確認する。ユノは記録に意図的な分割痕があると示す。

### 第2記録回収後

外部命令の痕跡を巡り、リゼは証拠保全を、ユノは装置の再現試験を主張する。

### ボス前

2人とも調査継続を認める。リゼは撤退条件、ユノは中央排出路の操作条件だけを伝える。

### 地域完了後

リゼは生態系回復を調査成果としてまとめる。ユノは先遣隊記録の欠損が人為的である可能性を示し、次地域への謎を残す。

## 7. 情報の分担

| 情報 | 主に説明する人物 |
|---|---|
| 調査目的、危険度、帰還 | リゼ |
| 異獣の行動変化 | 主人公の観察、リゼの確認 |
| 古代装置の構造 | ユノ |
| 先遣隊記録の欠損 | ユノ |
| 師匠の研究姿勢 | 主人公の回想、記録本文 |
| 兵器化計画の疑い | 記録から主人公が推論し、2人が別の根拠で補強 |
| 共鳴継承 | ユノが手順、リゼが倫理・生態面を確認 |

説明役を固定し、両者が同じ情報を順番に言い直さない。

## 8. 立ち絵と表情

縦切り版では各人物につき、胸上の基本立ち絵1枚と表情差分3種を候補とする。

### リゼ

- 基本：落ち着いた観察
- 警戒：眉と視線を強める
- 安堵：口元だけをわずかに緩める

### ユノ

- 基本：計測端末へ視線を落とす
- 興味：目線を上げる
- 困惑：装置の値を疑う

大きな全身立ち絵や口パクは作らない。生態図鑑風の線画と研究所の紙・計測器UIに馴染ませる。

## 9. 音声

縦切り版では人物音声を収録しない。

- 発言開始時の小さなUI音だけ使用可能
- 人物ごとの疑似音声や意味のない発声は必須にしない
- 音声なしでも話者と感情を理解できる文章と表情にする

## 10. データ設計

```ts
type CharacterDefinition = {
  id: 'lize_ordo' | 'yuno_keil';
  displayName: string;
  roleName: string;
  portraitId: string;
  availableExpressions: string[];
};
```

イベント文章側では表示名を直書きせず、`speakerId` から参照する。名前変更時に全台詞を編集しなくてよい構成にする。

## 11. 採用しない構成

- チュートリアル専用の案内マスコット
- 研究主任を序盤の妨害役にする展開
- 記録技師を古代技術へ無条件に執着する人物にする展開
- どちらかを縦切り版内で裏切り者にする展開
- 師匠の代わりに全ての正解を教える人物
- 研究所内だけで長く続く人物会話

## 12. テスト項目

- 2人の担当情報が重複しすぎない
- 台詞だけでどちらが話しているか区別できる
- 初回出発前の会話が5発言以内に収まる
- 主人公の選択へ異なる短い返答が返る
- 重要な操作情報が人物会話以外でも確認できる
- 先遣隊記録より先に真相を断定しない
- 2人の意見差が善悪対立にならない
- 表情差分がなくても話者名と文章で意味が通る
- 名前を内部IDから変更してもイベントデータが壊れない
- ボス後に2人それぞれの視点で成果が整理される

## 13. 次に進めること

縦切り版を開始するための企画・技術・画面・導入仕様は十分に揃った。次は追加設定を増やすより、React＋TypeScript＋Viteの技術スパイクを作り、iPhone向けPWAの起動・保存・オフライン動作を実証することを推奨する。

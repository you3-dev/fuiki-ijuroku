# 『封域異獣録』育成システム

版：0.1  
更新日：2026年7月13日

## 1. 設計目標

- 種ごとの能力的な個性を維持する
- 同じ種でも役割を少し変えられる
- 共鳴継承による育成計画へ意味を持たせる
- 成長結果の引き直しを不要にする
- レベルアップのたびに能力ポイントを入力させない
- 能力値の無制限な世代インフレを防ぐ

## 2. レベル

- 縦切り版の上限：10
- 個人用MVPの上限候補：30
- 必要経験値はレベルごとのテーブルで管理する
- レベル低下は発生しない
- 共鳴継承で誕生した幼体はレベル1から開始する

## 3. 能力値の計算

各種は、能力ごとに基礎値と1レベルあたりの成長量を持つ。

```text
レベル能力値
＝ floor(
    (種の基礎値
     ＋ 種の成長量 × (レベル − 1))
    × 成長傾向補正
    × 個体差補正
  )
```

- 種の基礎値：レベル1の標準値
- 種の成長量：レベル上昇ごとの標準増加量
- 成長傾向補正：0.95〜1.12程度
- 個体差補正：0.97〜1.03
- 最終計算で小数点以下を切り捨てる

個体差補正は誕生・加入時に一度だけ決定し、レベルアップごとに再抽選しない。セーブデータには補正値または生成シードを保存する。

## 4. 経験値配分

戦闘終了時の配置と状態に基づいて経験値を配分する。

| 状態 | 獲得率 |
|---|---:|
| 前衛・戦闘可能 | 100% |
| 前衛・戦闘不能 | 50% |
| 控え | 75% |
| 研究所待機 | 0% |
| 戦闘終了後に加入 | 0% |

- 交代した個体は戦闘終了時の配置で判定する
- 経験値の端数は切り捨てる
- 協力要請で終了した戦闘も通常の戦闘経験値を与える
- 逃走では原則として戦闘経験値を与えない
- 調査イベント経験値は戦闘経験値と別に扱える

## 5. 成長傾向

| 傾向 | 補正案 | 代償案 |
|---|---|---|
| 強健型 | 生命 ×1.12 | 機敏 ×0.95 |
| 攻勢型 | 攻勢 ×1.12 | 抵抗 ×0.95 |
| 堅牢型 | 堅牢・抵抗 ×1.08 | 機敏 ×0.95 |
| 感応型 | 感応 ×1.12 | 攻勢 ×0.95 |
| 機敏型 | 機敏 ×1.12 | 生命 ×0.95 |
| 均整型 | すべて×1.00 | なし |

数値は縦切り版の初期案であり、プレイテストで変更できる。

### 野生個体

種ごとに出現可能な傾向を2〜4種類設定する。完全なランダムではなく、生息地点、希少個体、観察結果によって候補を絞る。

例：

- トモシゴケ：感応型、機敏型、均整型
- ヌマクグリ：強健型、堅牢型、均整型
- スミワタリ：感応型、堅牢型、均整型
- キリハネ：機敏型、感応型、均整型
- レキマトイ：攻勢型、堅牢型、強健型

### 共鳴継承個体

- 継承容量2を使い、提供元の傾向から1つを選べる
- 容量を使わない場合、幼体の種に適合する候補から選ぶ
- 世代数そのものによる能力補正は付けない

## 6. 個体差

- 各能力を0.97〜1.03の範囲で補正する
- プレイヤーへ細かな補正率を強調表示しない
- 図鑑または個体詳細では「やや高い」「標準」等の評価を表示できる
- 最大値を狙う再取得が必須にならない差に留める
- 希少個体は見た目や生態条件で特徴を出し、能力差を極端にしない

## 7. 能力値インフレの防止

- 提供元の現在能力値を幼体へ直接加算しない
- 世代数による恒久的な能力加算を行わない
- 強さの主な差を、技能、特性、成長傾向、編成相互作用へ置く
- レベル上限付近でも低レベル技能が完全に無意味にならない計算式を使う

## 8. 信頼度

縦切り版では段階0〜5の離散値とする。

### 上昇要因

- 調査へ同行する
- 前衛として戦闘を完了する
- 協力条件に沿った行動をする
- 戦闘不能にならず帰還する
- 適切な手当や生態イベントを選ぶ

### 用途

- 共鳴継承の実行条件
- 継承技能の記憶封印解除
- 個体詳細の生態反応
- 一部の観察・調査イベント

信頼度による直接のダメージ倍率はMVPでは付けない。

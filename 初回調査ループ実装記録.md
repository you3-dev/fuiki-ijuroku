# 『封域異獣録』初回調査ループ実装記録

更新日：2026年7月13日  
対象：`app/`  
判定：地点選択型の最小縦切りループとして成立。分岐先イベントと汎用戦闘は次工程

## 1. 今回の目的

「地点選択＋短い分岐調査」が本作の入口として機能するかを、説明資料だけでなく実際に操作できる状態で確認する。対象範囲は次の一周とした。

```text
研究所
→ 湿原入口の観察
→ 灰苔の浅瀬
→ 初回観察戦
→ スミワタリ加入・呼称入力
→ 観測櫓跡／沈み水路の分岐選択
→ 研究所へ帰還
```

分岐先の地点イベント、自由な行動選択を持つ汎用戦闘、敗北処理は今回の範囲外である。

## 2. 実装した内容

### 調査と戦闘

- 調査専用の全画面UIと地点データ
- 湿原入口を観察した後だけ灰苔の浅瀬を解放
- 警戒度60、汚染状態、汚染除去具1個から始まる固定チュートリアル戦
- 観察、濁り水圧への防御、汚染除去、静かな明滅、協力要請の順序制御
- `観察済み＋水圧防御済み＋汚染解除済み＋警戒度20以下＋鎮静`から協力可否を導出
- 条件達成後の協力要請は抽選なしで必ず成功
- 戦闘中にブラウザの戻る操作をしても調査画面へ戻す経路ガード

### 加入と分岐

- スミワタリを固定個体IDで前衛3枠目へ追加
- 任意の呼称を20文字以内で保存
- 研究報告を重複なく追加
- 異獣録を「調査済み 3 / 6」「スミワタリ・観察済み」へ更新
- 観測櫓跡と沈み水路を同時解放し、選択した経路を保存
- 加入済みの再出発では初回戦を繰り返さず分岐選択から再開

### 保存と復旧

- `GameSessionState` をスキーマ版2へ更新し、調査状態と戦闘ラウンドを同じスナップショットへ保存
- 旧スキーマ版1のIndexedDBセッションとJSONバックアップを版2へ移行
- 外側メタデータとセーブ本体の版が食い違うデータを拒否
- フェーズ、地点、加入フラグ、所有個体の相互整合を検査
- 協力成立後は重要保存の完了を待ってから呼称入力を有効化
- 保存失敗、再保存中、保存済みを調査画面で区別し、再保存ボタンを表示

## 4. 2人体制レビューで反映した事項

実装担当と独立レビュー担当に分け、正常系だけでなく保存失敗と不整合セーブを確認した。レビューから次を今回の実装へ反映した。

- 戦闘中の他画面への離脱をルートガードで防止
- 保存失敗を「保存中」と誤表示しない状態表示と再試行導線
- 加入成立後、重要保存完了前は呼称入力を出さない制御
- 準備未完了、前衛3枠目が空いていない状態では初回調査を開始しないルール
- 前衛・控えにまたがる固定個体IDの重複防止
- スキーマ移行、版情報不一致、加入と調査フェーズの相互整合テスト

## 5. 意図的な簡略化

- ルートは今回の小さな一周に合わせて `#/exploration` へ統合した。地域追加時に `#/exploration/:regionId` と戦闘専用ルートへ分離する。
- 初回戦はゲームルールを試す固定チュートリアルであり、HP、対象選択、行動順、敗北、敵AIを持つ汎用戦闘ではない。
- 異獣の図版は生態図鑑風イラストへ差し替える前の文字記号である。
- 分岐先は選択と保存までで、地点イベントは未実装である。

## 6. 次の推奨実装

次は「観測櫓跡」の1地点を完成させ、固定手順から汎用戦闘モデルへ広げる。

1. 味方3体と敵1体のHP、活性値、予定行動、対象を定義
2. 主人公支援と異獣行動を別枠で入力
3. 1ラウンド確定保存、敗北時の遭遇直前復帰
4. 観測櫓跡の短い分岐イベントと上流弁の復旧
5. プレイ時間と操作感を確認し、地点選択方式を継続するか判定

地点選択方式の評価が低い場合も、戦闘・地点データ・保存状態は流用し、表示層だけをタイルマップ比較試作へ差し替える。

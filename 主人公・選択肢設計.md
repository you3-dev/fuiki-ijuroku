# 『封域異獣録』主人公・選択肢設計

版：0.1  
更新日：2026年7月13日  
対象：縦切り版

## 1. 採用方式

主人公の発言は「半固定型」とする。

- 観察した事実、調査上の仮説、必要な確認は自動で短く発言する
- 価値判断、他者への返答、報告の優先順位はプレイヤーが選ぶ
- 選択によって戦闘能力、報酬、仲間化成功率を変えない
- 選んだ内容は会話の返答や後の短い言及へ反映する

主人公を完全な無言役にはせず、研究者として考えて行動する人物であることを示す。一方で、長い固定台詞によってプレイヤーの価値観を決めつけない。

## 2. 主人公が自動で示すこと

### 事実

```text
濾過膜の内側に、黒い沈殿が残っている。
```

見えているものを具体的に述べる。プレイヤーがまだ知らない設定を断定しない。

### 仮説

```text
敵意より、器官の痛みが行動を乱している可能性が高い。
```

「可能性」「らしい」「まだ断定できない」を使い、後の観察で修正できる余地を残す。

### 調査判断

```text
攻撃する前に、沈殿を安全に除去できるか試したい。
```

主人公の職業上自然な判断は自動で示す。ただし、プレイヤーの道徳性を評価する言い方にはしない。

### 師匠の教え

```text
「理解する前に分類するな」――あの人なら、まず反応を見たはずだ。
```

頻繁な決まり文句にはせず、地域内で1〜2回程度に抑える。師匠を正解を教える声として使わない。

## 3. プレイヤーが選ぶこと

### 価値判断

```text
[異獣の苦痛を先に報告する]
[施設異常の危険を先に報告する]
[先遣隊の痕跡を先に報告する]
```

どれも調査員として妥当な観点にする。善悪の正解が明らかな選択肢を並べない。

### 対人応答

```text
[慎重に確認してから答える]
[現時点の仮説を伝える]
[まだ断定できないと伝える]
```

選択肢は発言全文より、意図が分かる要約として表示する。選択後に主人公が自然な短い発言を行う。

### 約束と態度

```text
[異獣の状態を最後まで記録する]
[施設停止の原因を突き止める]
```

物語上の大きな約束は選択式にできるが、縦切り版ではどちらを選んでも地域調査の内容を失わせない。

## 4. 選択肢を使わない場面

- 観察できた事実の確認
- チュートリアル中の次操作
- 戦闘ラウンド中の時間を止める会話
- 協力条件が成立したかどうか
- セーブ、入手、解放などのシステム通知
- プレイヤーが選んでも実際には拒否できない行動

「調査へ行く／行かない」と選ばせた直後に強制出発するような偽の選択肢を作らない。

## 5. 選択肢の数

通常は2〜3個とする。

| 数 | 用途 |
|---|---|
| 2個 | 明確な優先順位や態度の違い |
| 3個 | 共感、慎重、探究など複数の妥当な観点 |
| 4個以上 | 縦切り版では原則使用しない |

選択肢の長さを極端にばらつかせず、詳細さによって正解を推測させない。

## 6. 選択結果の反映範囲

### 反映する

- 直後の相手の返答
- 主人公の次の短い発言
- 研究所ホームで表示する報告の順番
- 後のイベントでの1回程度の言及
- 調査記録に残る主人公の注記

### 反映しない

- 能力値
- 経験値や研究資源
- 協力要請の成功条件
- 共鳴継承の結果
- 重要な異獣録情報の取得可否
- 縦切り版のボス到達条件

選択肢は人格表現と物語の視点を変えるために使い、攻略上の正解探しにしない。

## 7. 選択の記録

数値化した善悪・性格メーターは作らない。選んだ選択肢IDだけを物語状態へ保存する。

```ts
type DialogueChoiceRecord = {
  eventId: string;
  choiceId: string;
  selectedAtRevision: number;
};
```

必要なイベントだけ選択IDを参照し、後の台詞を1〜2行変える。すべての選択を組み合わせた大規模な分岐管理は行わない。

## 8. 名前と呼び方

- 主人公名はプレイヤーが設定する
- 他者は通常、設定した名前＋役職または敬称で呼ぶ
- システム文では「主人公」ではなく名前を使う必要はない
- 名前が長くても会話欄を壊さない
- 名前を文中へ何度も埋め込まない
- 禁止語のオンライン共有がないため強い入力規制は不要だが、空文字と制御文字は許可しない

日本語以外の名前や記号を含む名前でも、助詞の自動変化を必要としない文章にする。

## 9. 一人称と性別表現

主人公の外見を選べるため、自動発言では一人称を省略できる自然な日本語を基本とする。

```text
良い例：まだ断定できない。もう一度、上流の反応を確認したい。
避ける例：僕／私の直感では、絶対にこれが原因だ。
```

- 固定の「僕」「私」「俺」を多用しない
- 性別を前提とした呼称や反応を避ける
- 他者からは名前または「調査員」と呼ばれる
- 恋愛関係や性別別の台詞分岐は縦切り版に入れない

一人称選択機能は、実際に文章上必要になった場合だけMVP後に検討する。

## 10. 発言の見せ方

### 自動観察

話者の顔を大きく出す会話枠ではなく、調査帳の短い記録札として表示する。

```text
観察
濾過膜の内側に黒い沈殿がある。
```

### 対人会話

話者名と本文を分けた会話欄を使う。主人公の外見設定がある場合は、選択した小さな肖像を表示してもよい。

### 選択肢

- 会話本文の下へ縦に並べる
- 選択前に全文を読める
- 時間制限を設けない
- 色だけで態度を分類しない
- 選択確定前に誤タップしにくい間隔を取る

## 11. 導入部での適用例

### 灰苔湿原への出発前

自動観察：

> 先遣隊の最終記録は湿原入口で途切れている。まずは同じ経路を確認する。

選択肢：

```text
最初に確かめたいことは？

[異獣の異常行動]
[浄化施設の停止]
[先遣隊の足取り]
```

この選択は研究所ホームの現在目的に添える副題と、帰還時の報告順へだけ反映する。

### 湿原入口

自動発言：

> 水面だけが揺れている。風ではない。

> トモシゴケの苔が灰色へ変わった。水へ触れる前に周囲を記録しよう。

ここでは選択肢を出さず、すぐ地点観察を操作可能にする。

### スミワタリ加入後

自動観察：

> 濾過膜の濁りは消えた。呼吸も安定している。

選択肢：

```text
研究所の記録には、何を最初に残す？

[苦痛の原因と除去手順]
[警戒度が下がった過程]
[遺跡波動との関連]
```

選択後、該当する観察記録を先頭へ表示する。協力条件自体はすべて異獣録へ登録する。

## 12. ボス戦での適用

ニゴリグイ戦では、戦術上必要な情報を選択肢にしない。

自動観察：

> 本体を傷つけても、排出できない汚染が戻るだけだ。先に中央排出路を復旧する必要がある。

戦闘終了後の価値判断だけを選択式にする。

```text
今回の浄化を、どう報告する？

[異獣との共同作業として記録する]
[施設の危険性を中心に記録する]
[外部からの制御痕跡を中心に記録する]
```

報告の並びと関係者の短い反応を変えるが、記録3個や地域完了報酬は変えない。

## 13. データ設計

```ts
type DialogueChoice = {
  id: string;
  intentText: string;
  spokenText: string;
  followupLines: EventLine[];
  addFlags?: string[];
};

type ChoiceEvent = {
  id: string;
  prompt: string;
  choices: DialogueChoice[];
  gameplayNeutral: true;
};
```

`intentText` は選択ボタンに表示する意図、`spokenText` は選択後に主人公が実際に話す短い文とする。

## 14. セーブと再読込

- 選択確定時に選択肢IDを保存する
- 保存完了後に相手の返答へ進む
- 再読込時は同じ選択肢を再提示しない
- 表示途中で閉じた場合は選択後の返答から再開できる
- 選択肢IDを変更するデータ移行では旧IDを対応付ける
- 会話選択を使って報酬処理を直接実行しない

## 15. テスト項目

- 主人公が観察事実を短く自動発言する
- 価値判断だけに2〜3個の選択肢が出る
- どの選択でもゲーム進行と報酬が同じである
- 選択後の返答と報告順が変わる
- 名前や外見に依存せず文法が成立する
- 一人称を省略しても不自然でない
- 選択途中の再読込で二重記録されない
- 選択肢の全文を文字拡大状態でも読める
- 選択肢を色なしで区別できる
- 選択内容が後の短い言及へ反映される

## 16. 次に決めること

研究所で主人公と会話する常駐人物を決める。縦切り版では、調査判断を管理する研究主任と、古代装置・記録復元を担当する技師の2人に絞る構成を推奨候補とする。

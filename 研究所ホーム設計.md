# 『封域異獣録』研究所ホーム詳細設計

版：0.1  
更新日：2026年7月13日  
対象：縦切り版／iPhone縦画面

## 1. 画面の役割

研究所ホームは、プレイヤーがアプリを開いたときに「現在の目的」「調査隊の状態」「次に押す場所」を短時間で理解するための基準画面である。

メニューを均等に並べるのではなく、現在の調査を最上位に置く。

### この画面で答える問い

1. 今は何を調べているのか
2. 次はどこへ行けばよいか
3. 調査隊は出発できる状態か
4. 前回の調査で何が増えたか
5. 研究所で利用できる機能は何か

## 2. 基本ワイヤーフレーム

```text
┌────────────────────────┐
│ 辺境研究所       ● 保存済み │  A. 上部バー
├────────────────────────┤
│ 現在の調査                 │
│ 灰苔湿原                    │
│ 上下流弁の状態を確認する      │  B. 調査目的カード
│ 記録 1/3　復旧 0/2          │
│ [ 灰苔湿原へ出発 ]           │
├────────────────────────┤
│ 調査隊                      │
│ 前衛                        │
│ [灯苔][沼潜][空き]           │
│ 控え                        │  C. 編成要約
│ [異獣][空き][空き]           │
│ 推奨：あと1体で前衛3体         │
│ [編成を確認]                 │
├────────────────────────┤
│ 新しい研究報告  2            │
│ ・キリハネの行動を記録        │  D. 更新要約
│ ・先遣隊記録01を復元          │
│ [報告を確認]                 │
├────────────────────────┤
│ 研究設備                     │
│ [共鳴継承] [資料・所持品]      │  E. 設備
│ [設定・保存]                 │
├────────────────────────┤
│  研究所  │  編成  │  異獣録  │  F. 下部タブ
└────────────────────────┘
```

## 3. A：上部バー

### 表示内容

- 画面名「辺境研究所」
- 保存状態
- 保存失敗時の詳細導線

通常時は情報を増やしすぎない。プレイヤー名、研究資源、設定アイコンをすべて上部へ並べない。

### 保存状態

| 状態 | 表示 |
|---|---|
| 保存済み | 小さな記録済み印＋「保存済み」 |
| 保存中 | 回転しない穏やかな進行表示＋「記録中」 |
| 再試行中 | 「保存を再試行中」 |
| 保存失敗 | 錆赤の固定表示＋「保存できていません」 |
| オフライン | 保存状態とは別に「オフライン」 |

オフラインであってもIndexedDBへ保存できている場合があるため、「オフライン」と「保存失敗」を同じ警告にしない。

## 4. B：現在の調査カード

研究所ホームで最も強いカードとする。

### 表示要素

- 地域名
- 現在の調査目的
- 短い状況説明
- 収集記録や復旧装置の進捗
- 出発または再開ボタン
- 出発できない場合の理由

### 状態別の主要ボタン

| 状態 | ボタン | 動作 |
|---|---|---|
| 初回 | `灰苔湿原へ出発` | 編成確認後、調査開始 |
| 調査中断後 | `調査を再開` | 最後の確定地点へ戻る |
| ボス解放前 | `浄化施設へ向かう` | 最終地点の調査確認へ |
| 地域完了後 | `調査報告をまとめる` | 完了報告と異獣録更新 |
| 編成不足 | `編成を整える` | 編成画面へ移動 |

出発不能のボタンを押せない灰色にするだけでなく、「行動可能な前衛がいません」など解決方法を示す。前衛が1〜2体の場合は出発を禁止せず、推奨編成との差として案内する。

## 5. C：編成要約

前衛3体と控え3体を、小さな個体札として表示する。

### 個体札の表示

- 一覧用アイコン
- 呼称または種名
- レベル
- HPの現在状態
- 戦闘不能、共鳴休止など重要状態

能力値や全技能は表示しない。詳しい確認は編成画面へ移す。

### 編成検査

調査開始前に次を検査する。

- 行動可能な前衛が1体以上いる
- 同じ個体が複数枠にいない
- 戦闘不能の個体だけで構成されていない
- 保存データ上に存在する個体IDである

前衛が3体未満の場合や役割が偏っている場合は警告しても出発を禁止しない。「前衛に空きがあります」「回復技能を持つ個体がいません」のように助言として表示する。

## 6. D：新しい研究報告

前回から増えた情報を最大3件まで要約する。

### 対象

- 新しく観察した種
- 調査段階が上がった種
- 判明した協力条件
- 先遣隊の記録
- 新しい触媒
- 地域目的の更新

情報が4件以上ある場合は件数を示し、「報告を確認」から一覧を開く。確認済みの更新は目立たせないが、異獣録自体から消さない。

報告を読むことは報酬受取条件にしない。通知を消すために何度も画面を開かせる作りを避ける。

## 7. E：研究設備

### 共鳴継承

- 条件を満たす個体がいない場合も設備名は表示する
- 利用不能理由を短く表示する
- 新しい組合せが可能な場合だけ小さな印を付ける
- 古代発光色は設備の状態灯に限定する

### 資料・所持品

縦切り版では研究素材、触媒、先遣隊記録を確認する簡易画面への入口とする。常設タブにはしない。

### 設定・保存

- 音量
- 動きを減らす設定
- JSONバックアップ
- JSON復元
- PWA更新
- セーブ初期化

初期化は研究所ホームから直接実行せず、設定内の複数段階確認を必要とする。

## 8. F：下部タブ

研究所タブを選択状態にする。

- アイコンと名称を併記
- 研究所を再度押すと画面先頭へ戻る
- 異獣録の未確認更新だけ件数または点を表示
- 編成の警告は出発不能など重要な場合だけ表示
- 保存失敗の警告をタブバッジへ流用しない

## 9. 初回プレイ時の表示

初回は情報量を抑え、調査開始までの流れを明確にする。

```text
1. 主人公設定完了
2. 研究所ホームへ到着
3. 現在の調査カードで灰苔湿原を紹介
4. 初期同行2体と空き枠を編成要約に表示
5. 2体編成のまま灰苔湿原へ出発
6. 灰苔の浅瀬でスミワタリを確定加入させ、前衛3体を完成
```

前衛は最大3体で空き枠を許可する。初回調査ではトモシゴケとヌマクグリの2体を必須とし、灰苔の浅瀬でスミワタリを確定加入させる。詳細は [導入・初回仲間化.md](./導入・初回仲間化.md) を参照する。

## 10. 帰還時の表示

調査から戻った直後は、更新要約を現在の調査カードの次へ一時的に繰り上げる。

### 帰還要約

- 調査した地点
- 観察した異獣
- 加入した個体
- 入手した研究素材
- 異獣録の更新
- 次の調査目的

詳細な戦利品一覧を毎回強制表示せず、重要な初回発見を中心にする。

## 11. 地域完了後の表示

ニゴリグイ鎮静後は現在の調査カードを完了報告へ切り替える。

- 灰苔湿原の浄化状態
- 回収した先遣隊記録3個
- 協力関係を結んだ種
- 未解決の観察項目
- 縦切り版のプレイテスト開始導線

次地域が未実装でも、単に「続きはありません」とせず、研究報告をまとめる形で区切る。

## 12. エラーと空状態

| 状態 | 表示 |
|---|---|
| 所有異獣0体 | セーブ不整合として通常進行を止め、復旧案内 |
| 現在目的なし | 完了報告または次の記録整理を表示 |
| 新情報なし | 更新カードを小さくするか非表示 |
| 背景画像失敗 | 紙地と文字だけで全操作を継続 |
| 保存失敗 | 最新状態を保持し、再試行とJSON退避を表示 |
| 更新待ち | 安全な研究所でのみ適用操作を有効化 |

## 13. 必要なデータ

```ts
type LaboratoryHomeViewModel = {
  saveStatus: SaveStatus;
  networkStatus: NetworkStatus;
  currentObjective: ObjectiveSummary;
  expeditionAction: ExpeditionAction;
  partySummary: PartySlotSummary[];
  partyWarnings: PartyWarning[];
  researchUpdates: ResearchUpdateSummary[];
  resonanceAvailability: ResonanceAvailability;
  pendingAppUpdate: boolean;
};
```

画面コンポーネント内で複数のドメイン状態を直接組み立てず、表示用セレクターから `LaboratoryHomeViewModel` を作る。

## 14. 部品分割

```text
LaboratoryHomePage
  AppHeader
    SaveStatusBadge
    NetworkStatusBadge
  CurrentObjectiveCard
    ProgressMarks
    PrimaryActionButton
  PartySummaryCard
    CreatureMiniCard × 6
    PartyWarningList
  ResearchUpdatesCard

```

編成、異獣録、探索でも再利用する部品は `features/laboratory` の内側へ閉じず、共通UIとして切り出す。ただし、再利用の見込みだけで細かく分割しすぎない。

## 15. 操作と保存

| 操作 | ゲーム状態変更 | 保存 |
|---|---:|---:|
| 研究所を表示 | なし | なし |
| 調査開始 | 調査状態を作成 | 重要保存後に遷移 |
| 調査再開 | なし | なし |
| 編成画面を開く | なし | なし |
| 報告を確認済みにする | 表示フラグのみ | 通常保存 |
| 共鳴継承画面を開く | なし | なし |
| PWA更新を適用 | 起動状態のみ | 最新状態の保存完了後 |

## 16. テスト項目

- 起動後、現在目的と主ボタンをスクロール前に確認できる
- 初回、調査中、帰還直後、地域完了後の各状態を表示できる
- 編成不足の理由と解決先が分かる
- 前衛3体＋控え3体を画面内で区別できる
- 新しい研究報告が最大3件に整理される
- オフラインと保存失敗を区別できる
- 背景画像がなくても操作できる
- 文字を拡大しても調査開始と下部タブへ到達できる
- 重要保存完了前に戦闘や探索へ遷移しない
- 未実装の次地域がなくても縦切り版の完了を理解できる

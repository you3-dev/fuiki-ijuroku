# 『封域異獣録』戦闘UX再設計

版：0.1（実装前設計）  
更新日：2026年7月15日  
対象：スミワタリ初回接触、観測櫓跡の汎用戦闘、沈み水路、浄化施設中枢  
判定：設計案。実装とプレイテストは未着手

## 1. 目的

実機プレイで判明した次の問題を解消する。

- スミワタリが、何をした結果として仲間になったのか分かりにくい
- 汎用戦闘へ入ると、次に何を操作すればよいか分からない
- 討伐ではなく調査・協力を目指す戦闘なのに、HPや行動入力が先に目へ入る
- iPhone縦画面で、目的、入力、結果が離れすぎている

この改修では戦闘ルールを作り直すのではなく、既存のルールをプレイヤーが理解できる順序へ並べ直す。

## 2. 現状分析

### 2.1 初回接触と汎用戦闘が別の操作体系になっている

スミワタリ戦では、画面が示す一つの行動を順番に押す。一方、直後の観測櫓跡では、次を同時に入力する。

- 主人公の調査支援
- トモシゴケの行動
- ヌマクグリの行動と、必要なら対象
- スミワタリの行動と、必要なら対象
- ラウンド実行

初回接触で覚えたのは「観察→防御→浄化→鎮静→協力」という固有手順であり、「目的に合わせて1ラウンド分の作戦を組む方法」ではない。この飛躍が最大の原因である。

### 2.2 初回汎用戦闘の情報量が多すぎる

観測櫓跡では、協力条件4個、主人公支援、味方3体のHP・活性・行動、対象、敵HP、警戒度、霧の残り時間、ログが同時に表示される。最大で4〜7個の入力判断が必要になる。

さらに、初期状態の「支援なし＋全員防御」でもラウンドを実行できるため、ルール上は正しいが調査が進まない行動を最初から選べてしまう。

### 2.3 成功条件の因果が見えない

「鳴き声観察」「周期を返す」「鎮静」「警戒20以下」は表示されるが、次が明示されていない。

- どの順番で達成するのか
- どの行動がどの条件を進めるのか
- 観察時に攻撃すると失敗すること
- キリハネのHPを0にすると調査失敗になること

### 2.4 結果が操作から離れている

実行結果は入力欄の下にある長い戦闘ログへ表示される。iPhoneでは、押した行動と結果を見比べにくい。保存機構の文言である「行動開始・ラウンドを確定保存」も、ゲーム内の目的を説明していない。

## 3. 採用する方式

### 3.1 比較

| 方式 | 内容 | 判定 |
|---|---|---|
| 全入力を常時表示 | 現在と同じく、支援と3体分の行動を毎回選ぶ | 初回には負荷が高すぎる |
| おすすめ作戦＋任意の個別編集 | 状況に合う隊全体の作戦を一つ示し、必要なら詳細を開く | **採用** |
| 完全な固定手順 | 正解ボタン以外を選べなくする | 理解より追従になりやすい |

採用方式では、プレイヤーの主判断を「このラウンドで何を達成するか」に置く。主人公と前衛3体の具体的な予定は、おすすめ作戦としてあらかじめ組む。

プレイヤーは一つの主ボタンで実行できる。個別に試したい場合だけ「個別指示を変更」を開く。

### 3.2 1ラウンドの基本サイクル

```text
状況を見る
→ 今回の目的を一つ理解する
→ おすすめ作戦を確認する
→ 必要なら個別指示を変更する
→ この作戦で進める
→ 結果を最大3項目で確認する
→ 次の目的を見る
```

## 4. 画面構成

390×844を基準に、最初の表示範囲へ目的、おすすめ作戦、主操作を置く。

```text
┌────────────────────────┐
│ 霧中のキリハネ　R1     │
│ 保存済み        ⚙設定   │
├────────────────────────┤
│ 今回の目的  1/3         │
│ 鳴き声の周期を記録する  │
│ 攻撃せず、霧の中で聞く  │
├────────────────────────┤
│ おすすめ作戦            │
│ 主人公：鳴き声を観察    │
│ 前衛　：全員防御        │
│ 見込み：観察成功        │
│         警戒度 －10     │
│ [ この作戦で観察する ]  │
│  個別指示を変更 ▾       │
├────────────────────────┤
│ 協力まで ○ ○ ○ ○       │
│ 敵HP 68 ※0で調査失敗   │
└────────────────────────┘
```

### 4.1 表示の優先順位

1. 今回の目的
2. おすすめ作戦と予測
3. 実行ボタン
4. 協力条件の進行
5. 敵と味方の詳細状態
6. 個別指示と詳細ログ

HP、活性値、対象選択、戦闘ログは消さないが、最初から主役にはしない。

### 4.2 文言

| 現在 | 変更後 |
|---|---|
| 行動開始・ラウンドを確定保存 | この作戦で進める |
| 確定済みラウンドを再開する | 保存した作戦を実行する |
| 前ラウンドの記録 | 今回起きたこと |
| 汎用戦闘・第Nラウンド | 接触調査・第Nラウンド |

保存は重要だが、通常時はヘッダーの「保存済み」で伝える。主ボタンへ保存実装の用語を出さない。

## 5. 観測櫓跡の初回汎用戦闘

既存の3ラウンド成功経路を、そのまま3枚の作戦カードへする。戦闘状態と保存スキーマは変更しない。

### 第1ラウンド：鳴き声を調べる

- 今回の目的：霧の中の鳴き声を記録する
- 主人公：霧中の鳴き声を観察
- 前衛3体：全員防御
- 予測：観察成功、警戒度10低下、受ける被害を軽減
- 主ボタン：`この作戦で観察する`

結果表示：

- 鳴き声の周期を記録した
- 警戒度が60から50へ低下
- 防御で生態被害を軽減した

### 第2ラウンド：同じ周期を返す

- 今回の目的：同じ周期を返して敵意がないと伝える
- 主人公：鎮静音具
- トモシゴケ：静かな明滅
- ヌマクグリ：トモシゴケをかばう、または防御
- スミワタリ：最もHPの低い味方を回復、不要なら防御
- 予測：周期応答、鎮静、警戒度40低下
- 主ボタン：`同じ周期で応答する`

結果表示：

- キリハネが音へ応えた
- 鎮静状態になった
- 協力を求められる状態になった

### 第3ラウンド：協力を求める

- 今回の目的：キリハネへ協力を求める
- 主人公：協力要請
- 前衛行動：協力成立時は実行されないため、個別入力を隠す
- 予測：条件達成済み、必ず成功
- 主ボタン：`キリハネへ協力を求める`

## 6. 個別指示

「個別指示を変更」を開いた場合のみ、現在の3体分のセレクトUIを表示する。各行動には名前だけでなく、役割と結果を付ける。

例：

- 防御：被害を40%軽減・活性＋25
- 灯角突き：キリハネを傷つけ、警戒度＋10
- 静かな明滅：警戒度－20・鎮静
- 身代わり潜行：指定した仲間をかばう
- 澄み流し：指定した仲間を回復・汚染解除

対象候補が1体なら、既存仕様どおり対象選択を表示しない。無効な選択肢は、黙って無効化するだけでなく「活性が5不足」のように理由を表示する。

## 7. 行動前の予測

おすすめ作戦と個別指示のどちらにも、確定前の結果予測を表示する。

- 達成できる条件
- 警戒度や汚染値の方向
- 攻撃による調査失敗の危険
- 防御が必要な予告行動
- 活性不足

観察中に攻撃が含まれる場合は、主ボタンの直前へ次の警告を出す。

> 攻撃音で鳴き声が聞こえなくなります。この作戦では観察できません。

敵HPには、HPの意味を明記する。

> 生態HPを0にすると倒してしまい、今回の調査は失敗します。

## 8. 行動後のフィードバック

実行直後は入力欄ではなく、同じ位置へ「今回起きたこと」を表示する。

- 最大3項目
- 条件達成、危険の解消、重要な数値変化を優先
- 変化した条件だけ短く強調
- 続けて「次にすること」を一つ表示
- 完全な戦闘ログは「詳しい記録」として折りたたむ

初期実装では、既存の`lastLog`と現在状態から要約を導出する。新しい永続データは追加しない。

## 9. スミワタリ初回接触の修正

### 9.1 開始時に勝利条件を伝える

```text
調査目的
汚染で苦しむスミワタリを助け、協力できる状態にする。

この接触では攻撃する必要はありません。
```

現在の5段階は維持するが、「正解ボタン」ではなく因果を表示する。

| 行動 | 行動後に理解させること |
|---|---|
| 観察 | 攻撃しているのではなく、濾過膜の汚染で苦しんでいる |
| 防御 | 安全な処置時間を作った |
| 汚染除去 | 苦しみの原因を取り除いた |
| 静かな明滅 | 敵意がないことを光で伝えた |
| 協力要請 | 条件が整ったため、抽選なしで応じた |

### 9.2 協力要請前に結果を予告する

```text
協力を求められます

成功すると：
・スミワタリが前衛3枠目へ加わる
・浄化技能「澄み流し」を使えるようになる
```

### 9.3 加入結果で理由を振り返る

加入画面へ次を追加する。

```text
スミワタリが応じた理由
✓ 苦しみの原因を観察した
✓ 汚染を取り除いた
✓ 敵意がないことを伝えた

スミワタリは自ら調査隊についてきました。
```

これにより「ボタンを順に押したら仲間になった」から、「助けた結果、協力関係ができた」へ意味を変える。

## 10. 後続戦闘への適用

### 沈み水路

- 今回の目的を、汚染塊の浄化、野生個体の浄化、鎮静、安全な流路の確保に分ける
- 最初はおすすめ作戦を提示し、ここで初めて複数対象の変更を教える
- 「対象HPを0にすると生態系損傷」を行動前に常時表示する

### 浄化施設中枢

- 3段階それぞれに一つの目的を表示する
- 排出暴発の予告中は「全員防御」を他の作戦より上へ固定する
- 環境操作6個を一つのセレクトへ常時並べず、現在段階で意味のある操作を主作戦へ出す
- 過負荷、警戒度、排出路のうち、現在段階で重要な値を強調する

## 11. 実装構造

保存スキーマを変更せず、既存状態から画面案内を導出する。

```ts
type BattleGuidance = {
  objective: string
  explanation: string
  recommendedSupport: string
  recommendedPlans: Record<string, unknown>
  forecast: string[]
  warning?: string
  primaryLabel: string
}
```

共通化候補：

- `BattleObjectiveCard`：今回の目的
- `RecommendedPlanCard`：おすすめ作戦と予測
- `BattleResultSummary`：最大3項目の結果
- `AdvancedBattlePlanner`：既存の個別入力
- `BattleConditionProgress`：成功条件と次の未達条件

最初は観測櫓跡だけで成立を確認し、同じ表示骨格を沈み水路と中枢へ展開する。

おすすめ作戦を一操作で反映する方法は、次の実装時に比較する。

1. 既存の`setSupport`と`setPlan`を順に実行してから`commitRound`
2. 作戦全体を一括確定するコマンドを追加する

保存の途中状態を画面へ見せないため、2を推奨する。コマンド型は増えるが、保存状態の形は変わらない。

## 12. 受け入れ基準

### 理解

- 戦闘開始から10秒以内に、討伐ではなく何を達成する戦闘か説明できる
- 各ラウンドで、次に行う主操作が一つだけ明確に見える
- スミワタリ加入後、なぜ仲間になったか説明できる
- 基礎行動、防御、技能、主人公支援の違いを観測櫓跡終了時に説明できる

### 操作

- おすすめ導線だけで、観測櫓跡を既存仕様どおり3ラウンドで完了できる
- 個別指示を一度も開かなくても初回汎用戦闘を完了できる
- 個別指示を開けば、既存と同じ行動と対象を選べる
- 実行後、スクロール位置を大きく失わず結果と次目的を確認できる
- 390×844で、目的、おすすめ作戦、主ボタンへ迷わず到達できる
- 320px幅で横スクロールしない

### 安全性

- 攻撃で観察に失敗する場合、確定前に理由が見える
- 敵HPを0にすると調査失敗になることが確定前に見える
- 無効な行動には選べない理由が見える
- `phase: committed`から再開しても、同じラウンドを一度だけ解決する
- 既存セーブ、決定論的乱数、敗北・再試行、加入処理を維持する
- 保存スキーマを版上げしない

### 人による確認

- 実機で説明なしに観測櫓跡まで操作してもらう
- 最初の入力までの時間、戻り操作、誤タップ、止まった場所を記録する
- 各戦闘後に「今回の目的」「行動の理由」「何が変わったか」を質問する
- 回答できない箇所は説明文の追加ではなく、情報の順序と操作数を先に見直す

## 13. 実装順

1. 観測櫓跡へ「今回の目的」「おすすめ作戦」「結果要約」を追加
2. 既存の個別入力を折りたたみへ移動
3. スミワタリ戦へ調査目的、協力結果の予告、加入理由を追加
4. 390×844で新規セーブからキリハネ加入まで通し確認
5. 沈み水路へ複数対象用ガイドを展開
6. 中枢戦へ段階目標と暴発時の最優先作戦を展開
7. 実機で人による無説明プレイテスト

## 14. 今回は行わないこと

- 戦闘数値や成功条件の全面変更
- タイルマップ移動への変更
- 育成、交代、共鳴継承の追加
- 新しいセーブスキーマ
- 長い説明モーダルの追加

説明を増やすことより、「今見る情報」と「今決めること」を減らすことを優先する。

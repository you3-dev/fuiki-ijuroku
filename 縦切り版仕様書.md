# 『封域異獣録』縦切り版仕様書

版：0.1  
更新日：2026年7月13日  
対象：iPhone向けPWA／Windowsで開発

## 1. 目的

本縦切り版は、完成版の一部を高密度で再現し、次の仮説を実機で検証するための最初の完成品である。

1. 前衛3体の個別指示によるターン制戦闘が、iPhoneで快適に操作できる。
2. 観察・鎮静・協力要請が、単なる捕獲抽選より面白い。
3. 地点選択と短い分岐だけでも、生態調査としての探索感を作れる。
4. 非消失型の共鳴継承が、次の個体を育てる動機になる。
5. ホーム画面起動、オフライン動作、中断復帰、バックアップが安定する。

縦切り版の目的はコンテンツ量を示すことではない。主要システムを一周つなぎ、個人用MVPへ進む価値があるか判断することにある。

## 6. 研究所

縦切り版の研究所は、自由移動するマップではなく機能メニューとして実装する。

### 機能

- **調査へ出る**：調査地域を選択する
- **編成**：前衛、控え、研究所待機を変更する
- **異獣管理**：個体の能力、技能、信頼度、系譜を確認する
- **異獣録**：発見・観察済み情報を確認する
- **共鳴継承**：基体、共鳴相手、触媒、継承内容を選択する
- **記録庫**：先遣隊の記録を読む
- **設定**：音量、演出速度、バックアップ、復元、診断情報

## 7. 分岐調査

### 灰苔湿原

古代水質浄化施設を中心に広がる、霧と苔に覆われた湿地。黒い沈殿物による水質悪化と異獣の興奮を調査し、施設と共生する地域ボスを鎮静する。

主な地点候補：

- 湿原入口
- 灰苔の浅瀬
- 沈み水路
- 観測櫓跡
- 濾過樹群
- 浄化施設中枢

詳細は [灰苔湿原.md](./灰苔湿原.md) を参照する。

### 確定した分岐

```text
湿原入口
   ↓
灰苔の浅瀬
   ├──────────┐
   ↓              ↓
観測櫓跡       沈み水路
   └─────┬────┘
         ↓
      濾過樹群
         ↓
   浄化施設中枢
```

- 観測櫓跡で上流弁を復旧する
- 沈み水路で下流弁を復旧する
- 両方の弁と濾過樹群の調査が完了すると中枢を解放する
- 1回の調査では観測櫓跡か沈み水路のどちらか一方を通る
- ボス到達まで最低2回の通常調査を必要とする

### 地域構造

地域はノードと接続線からなる有向グラフとして定義する。

```text
開始
 ├─ 観察地点 ─ 戦闘地点 ─┐
 └─ 採取地点 ─ 休息地点 ─┼─ 最深部
                           └─ 希少地点
```

### ノードデータ

各地点は少なくとも次を持つ。

- 地点ID
- 表示名
- 地点種別
- 接続先
- 解放条件
- 背景ID
- 発生イベントID
- 遭遇テーブルID
- 報酬ID
- 一度限りか再発生可能か

### 中断

- 地点選択前に自動保存する
- 地点イベント終了後に自動保存する
- アプリを閉じた場合は最新の確定状態から再開する
- 任意帰還では取得済みの記録と一部資源を保持する
- 敗北時の損失は縦切り版では小さくする

## 8. 戦闘

### 編成

- 味方：前衛1〜3体＋控え最大3体。通常は前衛3体を推奨
- 敵：1〜3体
- ボス：1体＋環境装置または追加行動

### コマンド

- 技能
- 基礎行動
- 防御
- 交代
- 道具

主人公は味方個体の行動とは別枠で、1ラウンドに原則1回、観察または調査支援を行える。

詳細は [戦闘システム.md](./戦闘システム.md) を参照する。

### 活性値

- 各個体が0〜100の活性値を持つ
- 戦闘開始時は原則40
- 基礎行動で15〜20回復する
- 防御で20〜25回復する
- 一般技能は20〜30消費する
- 強力な技能は40〜60消費する
- 戦闘終了時に次の戦闘用の初期値へ戻す
- 活性値不足の技能は選択できず、必要量を表示する

### 入力

1. 現在行動可能な前衛1〜3体の行動と対象を選択する。
2. 予定行動を一覧表示する。
3. 任意の個体まで戻って修正できる。
4. 行動開始後、素早さと優先度で処理する。

### 入力短縮

- 前ラウンドと同じ
- 基礎行動一括
- 対象候補が1体なら選択省略
- 標準、高速、最速の演出速度

### 勝敗

- 敵全員を戦闘不能にすると勝利
- 出撃中の前衛と控えがすべて戦闘不能になると敗北
- 協力要請が成立し、敵対個体がいなくなった場合も戦闘終了

### 属性と生態状態

縦切り版では固定の属性相性表を使わない。種の系統タグは図鑑、共鳴継承、イベント条件に使用し、ダメージ倍率へ直接結び付けない。

戦闘中は次の生態状態と環境状態を組み合わせる。

| 状態 | 基本効果 |
|---|---|
| 湿潤 | 霧・泥に関係する技能の条件になる |
| 霧中 | 一部技能の対象・効果を変え、命中低下効果の発生源になる |
| 汚染 | ラウンド終了時の被害または弱体。浄化対象になる |
| 装甲 | 物理被害を軽減する |
| 露出 | 装甲解除後に付与され、物理被害を増やす |
| 鎮静 | 攻勢と警戒度を下げ、協力要請へ近づける |

同名状態は原則として重複せず、持続時間を更新する。詳細な持続時間と成功判定は戦闘仕様で定義する。

状態変化は技能の条件を満たせば原則として成功する。持続時間は次で決める。

```text
持続時間
＝ 基本2ラウンド
＋ floor((使用者の感応 − 対象の抵抗) ÷ 20)
```

- 最短1ラウンド、最長3ラウンド
- 使用者を必要としない環境効果はデータで固定する
- 完全耐性は能力値ではなく明示的な耐性タグで定義する
- 完全耐性と短縮結果は行動予測へ表示する

## 9. 仲間化

### 状態

- HP
- 警戒度
- 観察段階
- 協力条件

### 初見

正確な条件を伏せ、外見、鳴き声、行動ログでヒントを出す。観察コマンドで条件を段階的に明らかにする。

### 調査済み

戦闘開始時から警戒度と既知の協力条件を表示する。

### 成功条件

1. 必要な観察段階へ到達する。
2. 警戒度を規定値以下にする。
3. 個体または種の協力条件を満たす。
4. 主人公の「協力要請」を使用する。

条件達成後は原則成功する。対象を先に戦闘不能にした場合、その戦闘では協力要請できない。

## 10. 成長と信頼

### 能力値

| 能力 | 用途 |
|---|---|
| 生命 | 最大HP |
| 攻勢 | 物理技能の威力 |
| 堅牢 | 物理技能への防御 |
| 感応 | 生態技能、回復、浄化の威力 |
| 抵抗 | 生態技能への防御と状態異常への耐性 |
| 機敏 | ラウンド内の行動順 |

活性値は0〜100の戦闘リソースとして別に扱い、レベル成長する能力値には含めない。

命中と回避は独立能力にせず、技能、特性、状態変化で一時的に扱う。

### ダメージ

```text
ダメージ
＝ 技能威力
× (50 ＋ 攻撃側能力) ÷ (50 ＋ 防御側能力)
× 乱数補正
```

- 物理技能は攻勢と堅牢を使う
- 生態技能は感応と抵抗を使う
- 乱数補正は0.95〜1.05
- 最低ダメージは1
- 小数点以下は最後に切り捨てる
- 防御中は計算後の最終被害を60%にする
- MVPでは会心攻撃を実装しない
- 乱数は保存可能なシードから生成する

### 回復

```text
回復量
＝ 技能回復力
× (1 ＋ 使用者の感応 ÷ 100)
```

最大HPを超えて回復しない。端数は最後に切り捨てる。

### レベル

- 戦闘・調査完了で経験値を得る
- 縦切り版の上限候補はレベル10
- 共鳴継承の確認を早めるため、レベル条件は本編より低く設定できる

能力成長の詳細は [育成システム.md](./育成システム.md) を参照する。

### 成長傾向

- 強健型
- 攻勢型
- 堅牢型
- 感応型
- 機敏型
- 均整型

共鳴継承では継承容量2で成長傾向を選択できる。野生加入個体は種ごとの候補から決定する。

### 経験値配分

| 戦闘終了時の状態 | 獲得率 |
|---|---:|
| 前衛・戦闘可能 | 100% |
| 前衛・戦闘不能 | 50% |
| 控え | 75% |
| 研究所待機 | 0% |
| 戦闘終了後に加入 | 0% |

戦闘途中で交代した個体は、戦闘終了時に前衛なら前衛、控えなら控えとして扱う。経験値の端数は切り捨てる。

### 信頼度

- 調査同行
- 戦闘での活躍
- 適切な休息や手当
- 生態に合った行動

縦切り版では3〜5段階の離散値とし、複雑な内部数値を画面へ出さない。

## 11. 共鳴継承

### 入力

- 基体
- 共鳴相手
- 任意の触媒
- 継承特性
- 継承技能
- 成長傾向

### 種の決定

```text
基体の種・系統
＋ 共鳴相手の系統タグ
＋ 触媒タグ
＝ 幼体の種
```

結果は実行前に確定表示する。

### 継承容量

- 初期案：10点
- 種固有特性：容量外で必ず保持
- 継承特性：原則1個
- 継承技能：最大2個
- 成長傾向：提供元の候補から選択

### 提供元

- 消失しない
- 能力や信頼度を失わない
- 実行後、一定回数の探索が終わるまで再度提供元になれない
- 休止中も戦闘と探索には参加できる

### 縦切り版向け調整

短時間で機能を検証できるよう、共鳴継承の初回だけ研究所イベントによって条件と資源を緩和できる。本編用の標準条件とはデータで分離する。

## 13. 物語

### 導入

主人公は辺境研究所へ新人調査員として配属される。師匠である先遣調査隊長は半年前から消息不明となっている。

### 縦切り版で明かす情報

1. 先遣隊は地域の異獣を攻撃せず、協力を得て進んでいた。
2. 地域の凶暴化は遺跡装置の異常と関係している。
3. 師匠は研究所へ直接記録を送れず、複数地点へ分散して残していた。

### 終了時の引き

地域ボスを鎮静すると遺跡装置が一時停止し、師匠の記録が復元される。記録には次の封鎖地域と、研究所内部へ情報を渡さないよう警告する一文が残されている。

### ニゴリグイ戦

1. 体表の汚染塊2個を除去する。
2. 主人公の調査支援で中央排出路を2段階復旧する。
3. 澄み流しと静かな明滅で過負荷値・警戒度を下げ、鎮静状態にする。
4. 条件達成後に「浄化経路を接続」して成功する。

```text
過負荷値20以下
＋ 警戒度20以下
＋ 鎮静状態
＝ 浄化経路を接続可能
```

本体HPをゼロにするか味方が全滅した場合は調査失敗となり、ボス戦直前へ戻す。詳細は [灰苔湿原.md](./灰苔湿原.md) を参照する。

- 初回ユーザー操作後に音声を開始する

- 音量と消音設定を保存する

縦切り版では無音でもすべて操作でき、音声エラーで進行不能にならないことを必須とする。

## 21. プレイテスト判定

各項目を5段階で評価し、自由記述を残す。

| 評価対象 | 主な問い |
|---|---|
| 戦闘 | 3体へ指示する手間に見合う判断があるか |
| 仲間化 | 条件を発見する過程が面白いか、面倒か |
| 探索 | 地点選択だけでも未知を調べる感覚があるか |
| 図鑑 | 情報が増えることを報酬と感じるか |
| 共鳴継承 | 結果を考え、幼体を育てたいと思うか |
| 操作 | iPhoneで誤操作や読みにくさがないか |
| 継続意欲 | 次の地域や異獣を見たいと思うか |

探索評価が低い場合は、同じ地域データを使った1地域限定のタイルマップ試作を作り、地点選択版と比較する。

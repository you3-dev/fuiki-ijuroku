# 『封域異獣録』敵予兆表示 実装記録

更新日：2026年7月17日

## 目的

プレイヤーが防御や身代わり潜行を選ぶ前に、敵が次に誰を狙うのか、単体攻撃か全体攻撃かを理解できるようにする。

戦闘後のログで理由を知るのではなく、ラウンド開始前に判断材料を提示する。

## 表示形式

敵HPまたは調査対象の状態表示の直後へ、1つの短い予兆欄を追加した。

`敵の予兆　翅刃　→　ヌマクグリ　単体攻撃`

表示する情報は次の3点に限定する。

- 技または現象の名前
- 標的
- 単体攻撃・全体攻撃・汚染・準備行動の区別

ダメージ量は乱数や防御状態で変化するため断定しない。

## 戦闘別の規則

### 若いキリハネとの通常戦闘

- 奇数ラウンド：翅刃 → トモシゴケ
- 偶数ラウンド：翅刃 → ヌマクグリ

### スミワタリ加入後の澄み流し実戦

- 奇数ラウンド：汚染飛沫 → スミワタリ
- 偶数ラウンド：汚染飛沫 → ヌマクグリ

### 観測櫓跡

- 霧が残る奇数ラウンド：霧裂き → 前衛全体
- 霧が残る偶数ラウンド：翅刃 → トモシゴケ
- 霧の残りが0：霧包み・攻撃なし・霧を張り直す

### 沈み水路

- 汚染塊が残っている：濁流弾 → トモシゴケに加え、黒い沈殿 → 前衛全体へ汚染
- 汚染塊を浄化済み：濁流弾 → トモシゴケ

複数の現象が同じラウンドに発生する沈み水路では、「単体攻撃＋全体汚染」と1欄にまとめる。

## 身代わり潜行との関係

敵予兆は敵が本来狙う対象を表示する。身代わり潜行の予定は、ヌマクグリが守ろうとしている対象を表示する。

たとえば通常戦闘の第6ラウンドでは、次の2つが同時に見える。

- 敵の予兆：翅刃 → ヌマクグリ
- ヌマクグリの予定：トモシゴケを守る

対象が一致しないため、このラウンドでは身代わり潜行より別行動が適していると行動開始前に判断できる。

## UI方針

- iPhone幅では2段に分け、技名と攻撃種別、矢印と標的を整理する。
- 単体攻撃は淡い茶色、全体攻撃・全体汚染はやや強い赤、霧の張り直しは青緑で区別する。
- 予兆欄そのものは操作部品にせず、タップ数を増やさない。
- 読み上げ時は「敵の予兆、技名、標的、攻撃種別」を1つの領域名として伝える。

## 保存とロジック

- 戦闘計算は変更していない。
- セーブデータ形式は変更していない。
- 敵の行動規則と同じラウンド番号・霧残数・汚染値から予兆を生成する。

## 検証

- 敵予兆の規則テストを3件追加した。
- 型検査、Lint、自動テスト68件、公開用ビルドを通過した。
- 390×844表示で通常戦闘第6ラウンドを再開し、「翅刃 → ヌマクグリ・単体攻撃」と表示されることを確認した。
- 予兆欄は約49px高に収まり、標的名と攻撃種別に文字切れがないことを確認した。
- 画面全体に横スクロールが発生しないことを確認した。
- ブラウザの警告・エラーがないことを確認した。

公開用ビルドには従来どおり、JavaScriptチャンクが500kBを超える警告がある。動作を妨げるエラーではない。

## 次段階

- 選択中の防御・身代わり潜行が予兆に対して有効かを、ラウンド予定欄で短く判定する。
- 全体攻撃時に「全員防御」、単体攻撃時に「標的が防御」など、対応行動を1つだけ提示する。
- 予兆を見てから行動を変更する流れが自然か、実機プレイで確認する。
